# Toolchain ownership migration matrix

`TempleOS/` is excluded: it is reference material, not a source cohort. Statuses describe ownership, not how much code exists.

The private production compiler sends `continue` in a `do` loop to the condition through a patched trampoline. `/bin/feature25.cc` proves the behavior in the in-OS JIT. This fixes code generation without changing ownership. The shared hosted frontend retains initializer forests, linked addresses, and typed function bodies. The new IR operation lowers the unchanged `cemit_add_overflows` helper, and the object operation emits its first represented leaf functions beside static data through the shared ELF32 and x86 seams. All eight hosted Toolchain source gates parse completely. No normal OS C transform has moved to this path. A `continue` inside a `switch` nested in a loop remains a private-emitter limitation, and issue #25 stays open.

# Share a typed AST and linear IR across two language modes

CupidC will parse C mode and Cupid mode into a shared typed AST and lower both through a shared linear IR, with C semantics and Cupid-specific extensions kept distinct at the language boundary. This replaces direct parser-to-machine-code generation while avoiding two independent compilers, enabling semantic checking and optimization without forcing Cupid extensions into ordinary C.

ADR 0013 establishes the first shared semantic layer beneath that frontend: one immutable indexed type graph and fixed i386 layout operation. The graph is intentionally parser-neutral and does not expose the private kernel parser or its direct code-generation state. ADR 0014 consumes ADR 0012's typed preprocessing tape, applies the contracted declaration and language-mode legality, constructs that graph, completes i386 layout, and now publishes immutable postorder function-body tables through the same transactional frontend operation. The unchanged FAT16 closure still reproduces the manual ABI oracles, while unchanged `debug.h` proves compound/expression statements, binding and parameter references, ordinary strings, fixed-argument prototyped calls, and explicit semantic conversions. Focused contracts additionally prove pointer arithmetic/subscripts, one-evaluation compound/update nodes with distinct result and computation types, and typed C11 `for` statements. A `FOR` node uses `CTOOL_C_AST_NONE` when its initializer is omitted and otherwise names a present expression or declaration statement; it also retains an optional scalar condition after the ordinary default conversions, an optional converted iteration expression, and one required body statement; its initializer and body precede it in statement postorder. Null expression statements retain `CTOOL_C_AST_NONE`, while `BREAK` and `CONTINUE` are targetless leaves whose nearest target remains a lowering decision. One loop scope covers a declaration initializer, both expression clauses, and the body, then expires; the parser distinguishes compound block items from statements so a declaration cannot masquerade as a bare loop body. Scalar automatic-object initializers retain a converted typed expression on each block binding, honor C's point of declaration, and accept nested single-element scalar brace wrappers with an optional trailing comma. Function definitions retain their own declared type, `inline`, storage, parameters, body, and dual location separately from the canonical entity summary. Floating loop conditions are valid but deferred until floating conversions exist, whereas aggregate and `void` conditions violate the scalar constraint. Conditional and comma expressions remain a deliberate expression boundary. Aggregate automatic, file-scope, and static-duration initializers; the remaining control flow and expressions; floating and null-pointer-constant conversions; translation-unit finalization; linear IR; code generation; and production compiler ownership remain later gates.

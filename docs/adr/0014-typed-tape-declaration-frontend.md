# Build CupidC declarations directly from the typed preprocessing tape

CupidC declarations will live behind one freestanding `ctool_c_parse` operation. The operation borrows ADR 0012's immutable preprocessing-token tape plus an explicit C/Cupid language-mode request and publishes one immutable, job-owned translation-unit result. That result contains the ADR 0013 indexed type/member/parameter graph, the corresponding completed i386 layouts, canonical ordinary bindings, named tags, and parameter metadata. Bindings distinguish typedefs, objects, functions, and enumerators. A canonical binding retains the first declaration's storage spelling and dual location while its linkage and type describe the merged entity; a composite type is symbol-local and never mutates a shared typedef graph. Enumerator bindings retain their target-width integer bit pattern and evaluated signedness. Every public name and both its presumed and immutable physical source location are copied into the job arena; the caller's tape and request are borrowed only for the call.

This is a declaration frontend, not a text adapter around the private kernel compiler. It consumes the typed tape as the only lexical authority, including already-expanded spellings, source provenance, and per-token `#pragma pack` caps. A successful call invokes type layout before publishing the translation unit. Any parser, semantic, storage, limit, or layout failure zeros the result, preserves a structured diagnostic, and rewinds every allocation made since the operation mark. The input tape is never mutated. A same-job recovery contract parses a valid declaration immediately after a precise syntax failure without invalidating the earlier preprocessing result. A separate ownership contract builds a tape in caller storage, parses it, destroys and overwrites every borrowed token spelling and path, and then rereads binding, tag, member, and parameter names and dual locations from the published unit.

The first active-source proof parses the unchanged `/kernel/fs/fat16.h` closure under the exact checked `KERNEL_I386` preprocessing profile. The authoritative tape has 710 tokens: 169 from `/kernel/core/types.h`, 129 from `/kernel/fs/blockdev.h`, and 412 from `/kernel/fs/fat16.h`. It yields 45 external declarations (24 typedefs and 21 functions), 50 ordinary bindings including five enumerators, 12 anonymous records with 78 members, two anonymous enums with five enumerators, and 27 function types with 44 normalized parameters, of which three are abstract. No tag name is invented for an anonymous definition. The frontend-selected compatible type for both nonnegative active enums is unsigned `int`; source `bool` remains an enum typedef rather than being collapsed into `_Bool`.

The next active-source gate keeps four exact unchanged header closures under that same profile. `/kernel/core/kernel.h` is 569 tokens/69 canonical bindings and retains external `print` at its first declaration in `/kernel/cpu/isr.h:7`; `/kernel/cpu/irq.h` is 512/57 and retains the no-linkage `irq_handler_t` typedef from `isr.h:18`; `/kernel/lang/cupidscript.h` is 1,867/177 and retains `script_context_t` from `cupidscript_streams.h:13`; `/kernel/lang/shell.h` is 591/74 and retains its one external `shell_jit_program_is_running` binding at line 81. A Python contract derives the exact 152-header active non-Doom cohort from the checked audit and invokes the public header-sweep mode. Compatible redeclarations first moved the gate to 140 successes; semantic `packed`, `aligned`, and `noreturn` support now moves it to 146 and pins the exact six remaining roots. Three SMP roots stop at `/kernel/smp/percpu.h:36` on `_Static_assert`; `debug.h`, `ports.h`, and `cpu.h` stop at inline definitions. This is declaration compatibility evidence, not production ownership.

The source-driven layouts reproduce the existing manual ABI oracles. `key_event_t`, `keyboard_buffer_t`, `keyboard_state_t`, `timer_state_t`, `timer_measure_t`, and `block_device_t` are respectively `12/4`, `3076/4`, `5156/4`, `20/4`, `16/4`, and `24/4`. The packed FAT partition, MBR, boot-sector, and directory-entry records are `16/1`, `512/1`, `36/1`, and `32/1`; the runtime filesystem and file records are `36/4` and `24/4`. All 54 FAT member offsets are pinned, and exactly 36 FAT wire members carry pack cap one while the 42 natural members carry no cap. Replacing only the already-expanded token for `KEYBOARD_BUFFER_SIZE` from `256` with `2` produces `keyboard_buffer_t` at `28/4` and `keyboard_state_t` at `2108/4`. That mutation proves the parser neither reopens source files nor reconstructs the old manual graph.

The frontend owns the declaration-specifier subset contracted by this slice, declarator precedence, pointers, fixed and unspecified arrays, function prototypes, parameter array/function adjustment, records, unions, Cupid classes, enums, typedef lookup, the represented storage classes and qualifiers, bit fields, and their declaration-time language rules. `_Thread_local`, `inline`, `_Noreturn`, `_Alignas`, `_Atomic(type-name)`, and complex/imaginary type specifiers remain explicit pending syntax rather than being implied by that ownership claim. Its namespace contract keeps ordinary identifiers and tags separate, gives function prototypes their required tag/enumerator scope, rejects duplicate parameter names, and does not leak prototype-scope definitions into file scope. It rejects reserved C words in identifier positions and additionally reserves Cupid words only in Cupid mode, so `class` remains an ordinary identifier in C mode. Published file-scope binding metadata distinguishes no storage class, typedef, extern, and static plus none/internal/external linkage. Visible `static` followed by `extern`, and a static function followed by an unadorned declaration, retain internal linkage; an external declaration followed by `static`, or a static object followed by an unadorned external-linkage declaration, conflicts. File-scope auto/register are rejected; valid prototype-parameter register syntax is checked but parameter storage is not yet part of `ctool_c_parameter_t` and must be retained when definitions are added.

Compatible object and function declarations use one iterative, memoized type-relation operation. It collapses duplicate/equivalent qualification wrappers, keeps referent qualifiers distinct, pushes array-typedef qualification to the element type, removes top-level parameter `const`/`volatile`/`restrict` while preserving atomic type identity, and treats a same-scope typedef redefinition more strictly than an object/function compatibility check. Old-style function declarations compose with prototypes only when every prototype parameter survives default argument promotion and no ellipsis is present. Incomplete and fixed arrays compose to the fixed form; opposing refinements in separate function parameters create a new composite parameter slice. Canonical binding replacement is copy-on-write: `typedef int A[]; extern A x; extern int x[4]; extern A y;` completes only `x`, leaving `A` and `y` incomplete. Repeated-pair work is memoized and stored in checked buffers, so a 24-level two-parameter compatibility DAG stays bounded, while 512 iteratively parsed pointer levels prove the 256 source-syntax limit is not misapplied to derived type graphs.

The GNU declaration subset parses normalized `__attribute` and `__attribute__` lists at active record, member, typedef, object/function, prefix, and interposed placements. `packed` updates the record/member layout policy; record alignment raises record layout; member alignment raises member placement; a typedef alignment creates an exact `CTOOL_C_TYPE_ALIGNED` wrapper without mutating its record; and object/function alignment publishes a minimum entity alignment on the canonical binding. `noreturn` is a binding-level function effect, not a type-compatibility distinction, and compatible declarations merge it in either order. Aligned wrappers compare transparently as C types while asymmetric composite selection retains the strongest alignment recursively and repeated typedefs retain the maximum. When alignment and structural refinements occur at opposing wrapper depths, composition separates common outer qualifiers from the structural children, combines the children, and reapplies exact alignment and qualifiers in source-semantic order. Contracts pin incomplete-aligned/fixed-array composition, nested exact wrappers, top-level `const`, and `_Atomic` minimum alignment in both declaration orders. Default-promotion checks unwrap alignment, so an aligned `char` prototype still conflicts with an old-style declaration in either order. Cupid i386 assigns bare `aligned`/`aligned()` the target default 16 and deterministically takes the maximum of repeated alignment requests. Unknown attributes, disabled GNU mode, malformed groups, invalid alignment, wrong arity, wrong targets, and currently unrepresented parameter attributes receive transactional `CTOOL_C_PARSE_DIAG_ATTRIBUTE` failures rather than being skipped. Unchanged source contracts pin `process_t` at `656/16` with `fp_state` at byte 80, the two packed IDT records, all five packed loader records, the per-CPU body at `128/4` with its typedef at `128/64`, both packed/aligned E1000 descriptors, and the three panic functions' retained `noreturn` effect.

Declaration-time completeness and qualification are checked where C requires them rather than inferred from final graph completeness. Every syntactic array declarator requires a complete object element type when the array is constructed, before a typedef can publish it or a parameter array can adjust to a pointer; completing a referenced tag later cannot repair an invalid array. In strict C mode an array element also cannot recursively contain a flexible-array member. By-value record members have the same temporal completeness rule, while an incomplete type remains valid in a function prototype and an external array with a complete element and unspecified bound retains its incomplete array representation. Void objects, arrays, and members are rejected; the single unnamed `void` prototype parameter is normalized to an empty prototype; named or non-sole void parameters are rejected. Array parameters adjust to pointers while retaining element qualification, and abstract `int ()`/`int (void)` parameter types adjust to pointer-to-function without manufacturing names. Qualified array objects retain their array representation, and declarations that would return arrays or functions are rejected. Flexible-array eligibility counts both direct and anonymously promoted names, accepts a promoted-name-plus-final-flexible-array record, rejects a sole flexible member, rejects an anonymous enum as a member, diagnoses promoted-name collisions, and propagates the strict-C embedding restriction through containing unions without rejecting the permitted union construction itself.

Integer constant expressions currently implement the declaration subset required for enumerators, array bounds, and bit-field widths: target-typed integer literals and enumerators, parentheses, unary sign/complement, multiplicative, additive, shift, and bitwise operators. Evaluation uses explicit Cupid i386 integer ranks, widths, conversions, signed overflow rules, unsigned wrapping, truncating signed division/remainder, arithmetic signed right shift, and checked shift counts rather than host `uint64_t` arithmetic. Contracts cover mixed signed/unsigned enumeration values, signed-`int` and signed-`long long` compatible enum selection, invalid suffixes and literals, divide/remainder by zero, overflow, invalid shifts, and implicit-enumerator overflow. Relational, equality, logical, conditional, comma, cast, `sizeof`, and address expressions are not yet part of this grammar and must be added when a declaration cohort requires them.

Declaration, member, tag, parameter, compatibility, and composite-type storage uses checked job buffers rather than the private frontend's fixed tables. A synthetic translation unit with 256 typedefs and one 256-member record proves the old 16-typedef and 32-member ceilings are gone. Namespace and member lookup is currently linear; this is practical for the contracted flat scale but must become indexed while preserving the contracted scope ordering before the complete kernel, Doom, and hosted-tool corpus can make lookup cost predictable. Recursive flexible-array containment uses visited logical-type IDs plus a checked `types + members + 1` work-item budget. A 24-level two-edge shared DAG without a flexible leaf must parse successfully, while the equivalent flexible-leaf graph must be rejected; removing visited-node suppression makes the bounded positive case fail and is therefore caught mechanically. Recursively nested declarators, constant-expression parentheses, and records have one public limit of 256 levels. This exceeds the C11 minimum translation limits and deliberately converts pathological 3,000-level source syntax into `CTOOL_C_PARSE_DIAG_LIMIT` with full rollback instead of consuming the host stack. Type compatibility/composition is iterative and consumes checked storage instead of that syntax limit.

The public boundary is fail-closed. Null tape/request pointers, an invalid mode or Boolean, a nonzero tape with no token array, and invalid token pack metadata produce the exact invalid-request diagnostic without consuming input or arena state. A one-byte configured output limit produces `CTOOL_C_PARSE_DIAG_LIMIT`, while a second limit is set to the exact pre-composite type-vector size for two compatible opposing function prototypes: parsing reaches the composite declaration, fails its next type append as a limit rather than an internal error, zeros output, restores the arena, preserves the tape, and leaves an earlier unit valid. A third measured limit reaches a trailing aligned typedef and fails exactly when its wrapper would extend the type vector, with the same unit/tape/arena rollback and prior-unit lifetime guarantees. A syntactically valid zero-bound array separately proves that a downstream ADR 0013 layout error rolls back the entire outer parse operation. Object initializers, function bodies, and Cupid `#exe` markers each return the documented unsupported diagnostic rather than being ignored.

Adapting the private kernel parser was rejected because it re-lexes flattened text, loses counted spellings, columns, physical/presumed provenance, pack metadata, qualifiers, and stable type identity, silently truncates several fixed-size tables, and emits machine code while parsing. The first success-only FAT contract was also rejected as sufficient evidence: adversarial review exposed gaps in prototype scope, reserved identifiers, promoted flexible-array names, temporal completeness, qualifier/storage/type legality, abstract parameters, integer semantics, enum compatibility, and pathological nesting. Those became positive and transactional negative contracts. Final semantic review then found null-token-array pointer arithmetic, enumerator reuse based on initializer signedness instead of declared type, incorrect outer/prototype shadowing, rejected valid incomplete prototypes and external arrays, and incomplete recursive flexible-array propagation. Standards review additionally found exponential traversal over a shared flexible-array type DAG, incorrect union treatment, nested prototype lookup ordering, a Windows-only audit expectation, and position-dependent handling of typed Cupid `#exe` markers. The fixes use declared enumerator types, ordered prototype marks, visited type traversal with a checked work bound, host-suffix-aware audit assertions, and one unsupported-marker normalization at the parser diagnostic seam. A last construction-time regression proved that typedef, adjusted-parameter, and union-member arrays could still bypass incomplete/flexible element rules; four red counterexamples moved the constraint into array construction. The first redeclaration implementation was likewise rejected after review: it recursively compared types under the source-syntax limit, accepted merely compatible typedef redefinitions, rejected valid parameter/array qualifier spellings and old-style prototypes, revisited a shared type DAG exponentially, and completed shared typedef nodes in place. Iterative memoized relations, strict same-type mode, normalized parameter types, and symbol-local composite construction replace that design. Raw unsigned-64 arithmetic for every integer expression was rejected after it produced incorrect signed division, conversion, overflow, and enum-selection behavior. Unbounded source-syntax recursion was replaced by the deterministic public limit. Static analysis findings are fixed rather than suppressed.

This operation does not yet parse function bodies or object initializers, publish/finalize tentative or defined entity state, accept static assertions, the remaining active attributes (`section`, `weak`, `used`, `noinline`, `unused`, `naked`, and `target`), `_Thread_local`, `inline`, `_Noreturn`, `_Alignas`, `_Atomic(type-name)`, or complex/imaginary type specifiers, implement the remaining constant-expression grammar, represent Cupid `#exe` policy or other Cupid extensions beyond declaration keywords/classes, build a statement/expression typed AST, lower to IR, generate machine code, or emit objects. Unsupported syntax is diagnosed rather than skipped; tentative incomplete-array declarations remain in their parsed form because definition state is not represented yet. The request's mode and GNU-extension setting must currently match those used to produce the tape because preprocessing provenance does not yet carry that configuration.

This capability is hosted bootstrap evidence only. GCC or Clang still builds the frontend and its contract. The private in-kernel CupidC frontend and code generator are unchanged, no production OS artifact consumes the new translation unit, and no boot or runtime behavior is claimed. The next header frontier is `_Static_assert` with `sizeof` and equality, followed by inline/function-body AST work along ADR 0003. The remaining active GNU attributes and extended assembly remain required for the `.c` cohort. Only typed AST/IR and object lowering can let the kernel adapter replace its private preprocessing/parser/code-generation pipeline. `TempleOS/` remains excluded reference material.

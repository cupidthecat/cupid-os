# Build CupidC declarations directly from the typed preprocessing tape

File-scope object state lives in a public definition table beside the canonical binding table. A canonical binding still keeps the first declaration's storage spelling and source location. Each object definition separately keeps its binding index, definition type, storage, explicit or tentative kind, dual source location, and initializer root. A finalized tentative definition uses the merged binding type. A later definition therefore does not rewrite facts from the first declaration.

An explicit definition reuses the semantic static initializer forest that already serves block-scope static objects. Tentative declarations for one binding coalesce into one definition record as they are parsed. Translation-unit finalization gives that record the merged binding type and a destination-typed `ZERO` root. An incomplete external array is completed to one element at that point. An incomplete internal array or incomplete record receives a precise diagnostic. A declaration that only uses `extern` creates no definition unless it also has an initializer.

`CTOOL_C_INITIALIZER_ADDRESS_BINDING` represents the address of a linked file object or function. The leaf keeps the destination pointer type, a stable canonical binding index, and a signed i386 target-byte addend. The parser accepts `&file_object`, array decay, function decay, and pointer addition or subtraction by a represented integer constant expression after the normal assignment conversion checks. Subscripts use that same pointer-arithmetic path, so `&array[index]` retains the array binding and the target-scaled byte offset. Each intermediate addend must fit the signed i386 relocation range. The parser discards the temporary expression tree after extracting the semantic address. It does not treat the value stored in a pointer object as a link-time address. Automatic-object addresses, runtime subscripts, member addresses, pointer casts, and other deferred address forms still fail closed. The frontend itself does not allocate sections, serialize bytes, choose symbols, or emit relocations. ADR 0015 consumes the frozen result through `ctool_c_emit_object` and owns those policies for the represented static-only subset.

It covers declaration-only `extern`, explicit and tentative definitions in both declaration orders, repeated tentative declarations, later fixed-bound completion, a complete internal array followed by a compatible incomplete redeclaration, scalar, string, aggregate, object-address, array-decay, function-decay, array-element, and one-past-array roots, unresolved external object and function references, and mixed address leaves in one structure. Separate cases prove pointer subtraction, integer-first addition, and `&numbers[1] + -1`, which returns to the array base. A qualified pointer retains the valid addend `0x7ffffffc`, while an object-to-`void` conversion retains addend four. The next valid array element would require addend `0x80000000` and receives the signed i386 range diagnostic. An unevaluated overflowing pointer expression inside `sizeof` is ignored correctly, while the outer evaluated offset remains a valid binding address. The unchanged `kernel/gui/gui_themes.c` prefix proves seven complete 35-member theme definitions and two tentative state objects under the checked kernel profile. The unchanged `kernel/fs/ramfs.c` prefix proves the copied name and all 11 function addresses in `ramfs_ops`. Exact failures cover duplicate definitions, invalid declarator kinds, incomplete tentative types, runtime values, incompatible or automatic addresses, stored pointer values, scalar object values, deferred member, runtime-subscript, cast, and block-static addresses, relocation-range overflow, declaration conflicts, rollback, and same-job recovery.

The body grammar publishes compatible structure and union values through plain assignment, function return, automatic expression initialization, fixed arguments, and same-record conditional expressions. These paths reuse the existing assignment, call, conditional, implicit-conversion, block-binding, and return records. Aggregate lvalues receive ordinary lvalue conversion before their value flows into another context. Every converted aggregate source and destination must be a complete object type. A conditional aggregate result is not an lvalue. Plain assignment also applies C11's full modifiable-lvalue rule: the target cannot contain a direct or recursively nested `const` member or array element. An iterative type walk with visited logical IDs and a `types + members + 1` work bound checks that rule without following pointers or revisiting shared subgraphs. Aggregate initialization from one compatible expression is represented. Automatic arrays and structures accept lists with runtime expression leaves, narrow character-array strings, brace elision, trailing commas, implicit zero tails, direct designators, and unknown-bound completion from the greatest selected index. Block-static arrays and structures accept the represented constant lists with the same direct designators.

The `aggregate-values` contract covers structures, unions, calls, chained assignment, volatile destinations, fixed arguments, return and initialization of records with `const` members, exact incompatible-type diagnostics, incomplete call and conditional operands, direct and nested nonmodifiable records, const flexible-array members, and records whose members point to const objects. It also pins non-lvalue assignment and conditional results, constrained output, rollback, tape immutability, prior-result survival, same-job recovery, and visited-type scratch rewind. The rewind guard parses a 240-link assignment chain after introducing 600 distinct pointer types under a measured 128 KiB output ceiling; disabling the rewind reaches the public storage-limit diagnostic inside that chain. The hosted Toolchain gate completes all seven unchanged sources. `ctool.c` reaches 65 definitions, 1,012 statements, 5,981 expressions, 133 block bindings, and 33 initializers; `cupiddis.c` reaches 65/1,470/9,607/149/114; `cupidld.c` reaches 66/2,064/13,347/267/146 with one canonical label; `cupidobj.c` reaches 14/289/1,984/39/21; `cupidc_type.c` reaches 31/737/5,487/85/43; `cupidc_emit.c` reaches 30/726/5,374/107/45; and `cupidc_frontend.c` reaches 284/10,773/68,625/1,591/1,131.

Static aggregate initialization uses an immutable postorder forest. A string `ADDRESS` leaf owns the decoded bytes of one ordinary narrow string literal and keeps a zero addend. A binding `ADDRESS` leaf names a linked file object or function and may carry a checked signed target-byte addend. Neither form carries a host address, section offset, ELF symbol index, or serialized pointer bytes. An explicit null pointer constant uses a destination-typed `ZERO` leaf. A `LIST` node owns a slice of direct edges. An array edge names an element index, and a structure edge names the absolute ADR 0013 member index. Every child precedes its parent. Slices concatenate in list postorder, every initializer has exactly one list, file definition, or block-binding owner, and omitted subobjects have no edge instead of expanding into zero leaves. ADR 0015, not the frontend, owns static-data allocation and `R_386_32` relocation policy.

The static list parser follows C11 brace ownership for fixed arrays, direct unknown-bound arrays, and complete structures. It recurses through brace elision, treats an explicit nested close as a hard subobject boundary, completes each declaration from its greatest direct outer element index, skips unnamed bit-fields, and excludes a final flexible array from positional or designated initialization. Ordinary narrow strings copy bytes into character-array subobjects, create address leaves for compatible character or `void` pointers, and convert to one for `_Bool`. Named bit-fields use their declared integer type. Direct member and array designators select one immediate subobject, retain source order, and move the following positional cursor. Chained selectors, promoted anonymous members, duplicate overrides, unions, Cupid classes, floating constants, pointer casts, member addresses, runtime subscripts, and nested incomplete-array completion remain fail-closed boundaries.

The `aggregate-initializers` contract reproduces CupidLD's four string addresses and 16 target-converted bytes. It also pins copied character rows, adjacent string-literal concatenation, direct address roots, recursive brace elision, partial zero tails, independent completion of a shared incomplete-array typedef, named and omitted bit-fields, and an unknown outer array whose final row is partial. The separate `designated-initializers` contract pins sparse and out-of-order arrays, positional continuation, structures and named bit-fields, explicit and elided nesting, block-static constants and addresses, one source-derived ELF byte table, constrained output, rollback, and exact diagnostics for every deferred boundary. Exact aggregate negatives still cover excess or empty lists, runtime static leaves, incompatible pointer leaves, and a reached flexible member. Freeze checks payload exclusivity, string termination, exact child types, record membership and eligibility, selector uniqueness, slice concatenation, postorder, and single ownership before publishing the forest.

The body grammar publishes typed C11 `while` statements for the represented scalar-expression subset. `CTOOL_C_STATEMENT_WHILE` retains one required controlling expression after ordinary lvalue, array, or function conversion and one required body statement. The body precedes its loop node in statement postorder. Integer and pointer conditions are represented; floating conditions remain a precise deferred boundary, while aggregate and `void` conditions fail their C constraints. The parser enters both breakable and iteration contexts only while it parses the body, so nested `break` and `continue` statements stay targetless for later lowering without leaking their validity outside the loop.

The public contract covers integer, pointer, volatile lvalue, array, function, and conditional-expression controls; null and compound bodies; nested loops; and loop-jump context. Nine exact failures cover aggregate, `void`, and floating controls, a direct declaration body, a missing opening or closing parenthesis, an absent condition, and absent bodies before a closing brace or at end of file. The same contract reaches the 256-level syntax limit and a constrained-output failure while proving zero failed output, exact arena and tape rollback, earlier-result survival, and same-job recovery.

The body grammar also publishes typed C11 `do` statements. `CTOOL_C_STATEMENT_DO` retains one required scalar condition after the ordinary conversions and one required body. The parser publishes the body and its expressions before the condition and loop node, matching the source's post-test order. It keeps both loop contexts active only for the body. `break` and `continue` remain targetless, and later IR must send `continue` to the condition.

The `do-statements` contract covers integer, pointer, volatile lvalue, array, function, and conditional controls; null and compound bodies; nested post-test loops; body-before-condition postorder; and loop-jump context. Exact failures cover aggregate, `void`, and deferred floating controls, a direct declaration body, every missing grammar component, and comma expressions. The same mode reaches the 256-level syntax limit and a constrained-output failure while preserving the borrowed tape, the earlier unit, the arena mark, and same-job recovery.

The body grammar publishes typed C11 `switch`, `case`, and `default` statements. A `SWITCH` node retains its controlling expression after lvalue conversion and integer promotion, plus one required postorder body. Each `CASE` node retains a folded integer constant converted to that promoted controlling type and one postorder body. A `DEFAULT` node retains its body. The parser keeps one checked context per nested switch, so inner case values and default labels do not leak into an outer switch. Duplicate values are rejected after conversion. A switch enters the breakable context while parsing its body but does not enter the iteration context, so `break` is valid and `continue` still requires an enclosing loop.

The focused contract covers signed and unsigned narrow controls, a volatile unsigned control, an unsigned-compatible enum, and a narrow unsigned bit-field. It checks ordinary and escaped character labels, typedef-named integer casts, arithmetic labels, chained fallthrough labels, signed-to-unsigned conversion, the target's two's-complement result for an out-of-range signed cast, and nested switch isolation. Exact failures cover noninteger controls, labels outside a switch, duplicate converted cases and defaults, invalid constant expressions, missing syntax or bodies, direct declaration bodies, and `continue` without a loop. The same mode reaches the 256-level syntax limit and a constrained-output failure while preserving the borrowed tape, the earlier unit, the exact arena mark, and same-job recovery.

The body grammar now publishes right-associative C11 conditional expressions. `CTOOL_C_EXPRESSION_CONDITIONAL` owns three ordered postorder children: the condition after ordinary conversion, the nonzero arm, and the zero arm. The node never yields an lvalue. Integer arms use the Cupid i386 promotions and usual arithmetic conversions. Compatible pointer arms use a composite referent and the union of immediate referent qualifiers. Object/`void` pairs use an appropriately qualified `void` referent and retain explicit pointer conversions on the object arm. A pointer paired with a null pointer constant keeps the pointer type and adds a destination-typed `CTOOL_C_CONVERSION_NULL_POINTER` to the other arm. Array and function operands decay before this selection, compatible function pointers remain valid, and two `void` arms produce `void`. Matching structure or union arms retain their common record type. Cupid follows N2209 for atomic referents in this operator. Atomic-qualified `void` and floating arithmetic arms remain precise deferred boundaries.

The integer constant-expression grammar uses the same conditional precedence and right association in static assertions, enumerators, array bounds, bit-field widths, alignment arguments, case labels, and supported block-static integer initializers. It accepts ordinary narrow character constants and integer-target casts over represented constant operands. Casts narrow with Cupid i386 widths and then apply the target integer promotion used by the surrounding expression. Out-of-range signed casts use Cupid's two's-complement target result. A floating literal immediately following an integer cast receives the deferred-feature diagnostic. Other floating constant-expression operands remain outside this grammar. The grammar parses and checks both conditional arms, applies the common target integer kind, and suppresses divide, remainder, overflow, and shift faults only in the unselected arm. A selected fault still fails at its operator. Comma expressions remain outside both expression grammars.

The body grammar now publishes typed pointer equality and relational comparisons. The ordinary lvalue, array, and function conversions run before constraint checking, and every comparison produces target signed `int`. Equality accepts pointers to compatible types, including function and incomplete object types. It also accepts an object pointer paired with a qualified or unqualified `void` pointer and records the required object-to-void conversion as `CTOOL_C_CONVERSION_POINTER`. A pointer paired with a represented null pointer constant receives an exact destination-typed `CTOOL_C_CONVERSION_NULL_POINTER` on the null operand. Relational comparison accepts compatible object pointer types. In C11 an object type may be complete, incomplete, or `void`; function types remain outside that rule. Cupid follows [WG14 N2209's proposed correction](https://www.open-std.org/jtc1/sc22/wg14/www/docs/n2209.htm): comparison and subtraction accept atomic, qualified, or unqualified versions of compatible referents, including mixed atomic pairs.

The shared assignment-conversion seam uses the same pointer conversion kind for object pointers converted to or from qualified or unqualified `void` pointers. The destination referent must retain every qualifier present on the source referent, and N2209's assignment rule requires both referents to be atomic or neither to be. Atomic-qualified `void` remains outside ADR 0013's layout contract, so the valid both-atomic object/void form is a precise unsupported type boundary. Calls, returns, automatic initializers, and assignment expressions share one checked scalar or compatible-aggregate rule instead of reconstructing it later. Runtime values carry private integer-constant-expression form, value-validity, and target-value metadata. A represented integer constant expression with value zero qualifies, as does that expression cast to a pointer whose referent is unqualified `void` when the pointer itself is not atomic-qualified. The shared seam records the conversion as `CTOOL_C_CONVERSION_NULL_POINTER`. Nonzero values, lvalue-converted objects, calls, assignments, updates, typed pointer casts, selected arithmetic faults, and atomic-qualified casts do not qualify and receive input diagnostics. Pointer/integer relational comparison is always a constraint error.

The preprocessing tape remains the immutable lexical authority and preserves the source spelling of C11 bracket digraphs `<:` and `:>`. The frontend interprets those spellings through one canonical bracket comparison wherever its supported grammar expects `[` or `]`, including array declarators, subscripts, and the fail-closed `__builtin_offsetof` array-designator boundary. Canonical and digraph delimiters may be mixed without rewriting or mutating the borrowed tape.

Compound/update validation distinguishes represented C constraints from unimplemented arithmetic. Floating `*=`, `/=`, `+=`, and `-=` computation and floating prefix/postfix updates are valid forms but remain explicit unsupported boundaries until floating conversions and arithmetic AST exist. Floating `%=`/shift/bitwise compounds and aggregate compound/update operands instead receive input diagnostics because they violate C operand constraints. Compatible aggregate plain assignment publishes an ordinary assignment node with one raw designator and an unqualified record result.

CupidC declarations will live behind one freestanding `ctool_c_parse` operation. The operation borrows ADR 0012's immutable preprocessing-token tape plus an explicit C/Cupid language-mode request and publishes one immutable, job-owned translation-unit result. That result contains the ADR 0013 indexed type/member/parameter graph, the corresponding completed i386 layouts, canonical ordinary bindings, named tags, parameter metadata, file object definitions, source-ordered block bindings, one semantic initializer forest with its direct-edge table, and the initial immutable typed function-body AST tables. File bindings distinguish typedefs, objects, functions, and enumerators. A canonical file binding retains the first declaration's storage spelling and dual location while its linkage and type describe the merged entity; a composite type is symbol-local and never mutates a shared typedef graph. A separate object definition retains definition-local facts and owns its static initializer root. Enumerator bindings retain their target-width integer bit pattern and evaluated signedness. Every public name, initializer byte sequence, and presumed or immutable physical source location is copied into the job arena; the caller's tape and request are borrowed only for the call.

This is a typed-tape frontend, not a text adapter around the private kernel compiler. It consumes the typed tape as the only lexical authority, including already-expanded spellings, source provenance, and per-token `#pragma pack` caps. A successful call invokes type layout before publishing the translation unit. Any parser, semantic, storage, limit, or layout failure zeros the result, preserves a structured diagnostic, and rewinds every allocation made since the operation mark. The input tape is never mutated. A same-job recovery contract parses a valid declaration immediately after a precise syntax failure without invalidating the earlier preprocessing result. A separate ownership contract builds a tape in caller storage, parses it, destroys and overwrites every borrowed token spelling and path, and then rereads file and block bindings, tags, members, parameters, definitions, statements, initializer records and edges, expression names and locations, plus decoded string bytes from the published unit.

The frontend owns the declaration-specifier subset contracted by this slice, declarator precedence, pointers, fixed and unspecified arrays, function prototypes, parameter array/function adjustment, records, unions, Cupid classes, enums, typedef lookup, the represented storage classes and qualifiers, bit fields, and their declaration-time language rules. C11 `inline` is accepted only for declarations of function identifiers, repeated use is idempotent, non-function placements fail transactionally at the specifier, and compatible declarations OR one `CTOOL_C_FUNCTION_DECL_INLINE` summary bit into the canonical binding without changing its established C linkage. That summary is deliberately not definition policy: each function-definition record now retains its own declared type, `inline`, and storage spelling before external-inline finalization or emission. `_Thread_local`, `_Noreturn`, `_Alignas`, `_Atomic(type-name)`, and complex/imaginary type specifiers remain explicit pending syntax rather than being implied by that ownership claim. Its namespace contract keeps ordinary identifiers and tags separate, gives function prototypes their required tag/enumerator scope, rejects duplicate parameter names, and does not leak prototype-scope definitions into file scope. It rejects reserved C words in identifier positions and reserves Cupid words only in Cupid mode, so `class` remains an ordinary identifier in C mode. Published file-scope binding metadata distinguishes no storage class, typedef, extern, and static plus none/internal/external linkage. Visible `static` followed by `extern`, and a static function followed by an unadorned declaration, retain internal linkage; an external declaration followed by `static`, or a static object followed by an unadorned external-linkage declaration, conflicts. File-scope auto/register are rejected; valid prototype/definition parameter `register` syntax is checked and retained in `ctool_c_parameter_t`.

Compatible object and function declarations use one iterative, memoized type-relation operation. It collapses duplicate/equivalent qualification wrappers, keeps referent qualifiers distinct, pushes array-typedef qualification to the element type, removes top-level parameter `const`/`volatile`/`restrict` while preserving atomic type identity, and treats a same-scope typedef redefinition more strictly than an object/function compatibility check. Old-style function declarations compose with prototypes only when every prototype parameter survives default argument promotion and no ellipsis is present. Incomplete and fixed arrays compose to the fixed form; opposing refinements in separate function parameters create a new composite parameter slice. Canonical binding replacement is copy-on-write: `typedef int A[]; extern A x; extern int x[4]; extern A y;` completes only `x`, leaving `A` and `y` incomplete. Repeated-pair work is memoized and stored in checked buffers, so a 24-level two-parameter compatibility DAG stays bounded, while 512 iteratively parsed pointer levels prove the 256 source-syntax limit is not misapplied to derived type graphs.

The GNU declaration subset parses normalized `__attribute` and `__attribute__` lists at active record, member, typedef, object/function, prefix, and interposed placements. `packed` updates the record/member layout policy; record alignment raises record layout; member alignment raises member placement; a typedef alignment creates an exact `CTOOL_C_TYPE_ALIGNED` wrapper without mutating its record; and object/function alignment publishes a minimum entity alignment on the canonical binding. `noreturn` is a binding-level function effect, not a type-compatibility distinction, and compatible declarations merge it in either order. Aligned wrappers compare transparently as C types while asymmetric composite selection retains the strongest alignment recursively and repeated typedefs retain the maximum. When alignment and structural refinements occur at opposing wrapper depths, composition separates common outer qualifiers from the structural children, combines the children, and reapplies exact alignment and qualifiers in source-semantic order. Contracts pin incomplete-aligned/fixed-array composition, nested exact wrappers, top-level `const`, and `_Atomic` minimum alignment in both declaration orders. Declaration-time completeness and qualification are checked where C requires them rather than inferred from final graph completeness. Every syntactic array declarator requires a complete object element type when the array is constructed, before a typedef can publish it or a parameter array can adjust to a pointer; completing a referenced tag later cannot repair an invalid array. In strict C mode an array element also cannot recursively contain a flexible-array member. By-value record members have the same temporal completeness rule, while an incomplete type remains valid in a function prototype and an external array with a complete element and unspecified bound retains its incomplete array representation. Void objects, arrays, and members are rejected; the single unnamed `void` prototype parameter is normalized to an empty prototype; named or non-sole void parameters are rejected. Array parameters adjust to pointers while retaining element qualification, and abstract `int ()`/`int (void)` parameter types adjust to pointer-to-function without manufacturing names. Qualified array objects retain their array representation, and declarations that would return arrays or functions are rejected. Flexible-array eligibility counts both direct and anonymously promoted names, accepts a promoted-name-plus-final-flexible-array record, rejects a sole flexible member, rejects an anonymous enum as a member, diagnoses promoted-name collisions, and propagates the strict-C embedding restriction through containing unions without rejecting the permitted union construction itself.

The operation publishes the first body layer under ADR 0003 without adding a second parse seam. `ctool_c_translation_unit_t` carries immutable flat object-definition, function-definition, block-binding, initializer, initializer-element, statement, expression, statement-child, and expression-child tables beside the declaration graph. Function definitions name their canonical binding but separately retain the exact definition declarator type, source storage, `inline` flags, body, and dual location. Definition parameter expressions index the definition type's existing parameter slice. Each `ctool_c_parameter_t` retains `register`/none storage plus the adjusted definition-local object type, including top-level qualification; the parallel `graph.parameter_types` entry remains the unqualified normalized type used for function compatibility. This split prevents a `const` or `volatile` definition parameter from becoming modifiable or losing its load semantics merely because C ignores top-level parameter qualification when comparing function types. A declaration statement indexes its ordered slice of `ctool_c_block_binding_t`; each binding owns its object name, source storage spelling, semantic type, optional initializer-table index, and dual location. An absent automatic initializer uses `CTOOL_C_AST_NONE`. A block-binding expression still indexes the binding table directly, so storage duration does not create a second object identity. Statement and expression nodes are postorder: every child index is smaller than its parent, slices preserve source order, and inapplicable indices use `CTOOL_C_AST_NONE`. Closed lexical scopes are parser scratch rather than public state, so their source-ordered bindings survive for later IR without exposing scope stacks, token pointers, or recursive ownership.

The initializer forest records source semantics rather than serialized target sections. An automatic scalar or whole-record initializer retains an `EXPRESSION` root. Automatic array and structure lists use `LIST` roots with converted runtime `EXPRESSION` leaves, while character arrays use `STRING`. A supported static object has a `ZERO`, target-converted `INTEGER`, copied narrow-array `STRING`, string or binding `ADDRESS`, or recursive `LIST` record. An explicit static null pointer constant becomes a destination-typed `ZERO` record after its temporary expression AST has been classified and rewound. This also applies to explicit pointer leaves in aggregate lists. A string address owns decoded bytes and keeps a zero addend. A binding address names a linked file object or function and may keep a checked signed target-byte addend. Neither form invents a host address or ELF symbol. A list owns a source-ordered slice of direct array-element or structure-member edges. Each edge may come from a positional clause or a direct C11 designator. Children precede parents, slices concatenate in list postorder, each initializer has one list, file definition, or block-binding owner, and omitted subobjects remain implicit zero without an edge. Direct unknown-bound arrays complete from the greatest selected index plus one without changing a shared typedef. Storage duration comes from the owning definition or block binding. Freeze requires every selector to be in range and unique, accepts `ZERO` in static forests, and rejects it in automatic forests; runtime leaves follow the opposite rule. Keeping the old expression-root field could not represent this state without inventing runtime nodes, while eager byte serialization would put storage and relocation policy inside the frontend. ADR 0015 now assigns the represented static definitions to `.rodata`, `.data`, or `.bss`, writes their target bytes, and constructs direct object-symbol `R_386_32` relocations through a separate transactional operation.

The body grammar deliberately follows unchanged active cohorts. It accepts nested compound, declaration, expression, return, `if`/`else`, counted `for`, typed `while` and `do`, typed `switch`/`case`/`default`, `break`, and `continue` statements. Block declarations cover complete automatic objects with none, `auto`, or `register` storage and supported complete or incomplete-array static objects. Automatic objects accept scalar or whole-record expressions, fixed or direct unknown-bound array lists, complete structure lists, narrow character-array strings, and direct C11 member or array designators. Lists preserve runtime expressions, use brace elision, accept a trailing comma, and leave omitted members and elements implicitly zero initialized. Any represented compatible record-valued assignment expression may initialize one record child, including a call, member access, parenthesized value, conditional, or compatible union value. An unbraced non-string clause is parsed once. A compatible record value is published for that child; otherwise the same AST value is carried down brace elision to the first scalar leaf. Parsing each clause once prevents speculative parsing, duplicate AST publication, and a local rollback seam. String tokens descend before expression parsing so a nested character array keeps string-initializer semantics. Static objects accept implicit zero initialization, represented integer constants, copied narrow strings, direct narrow-string addresses, and recursive lists with direct designators for fixed arrays, direct unknown-bound arrays, and complete structures. A static `for` initializer remains unsupported. Comma-separated declarators, pointers, and types already expressible through the declaration subset remain valid. Empty and excess lists fail precisely. A direct designator selects one immediate array element or named structure member. Edges retain source order, the positional cursor resumes after the selection, and an incomplete array completes from the greatest selected index. An implicit brace-elided child leaves a following designator for the nearest explicit list. Chained designator paths, promoted anonymous-member designators, duplicate overrides, and positional union or Cupid class lists remain deferred for both storage durations. C's point of declaration is preserved by making a provisional binding at its future stable index visible immediately after the declarator and throughout its own initializer, then publishing the finalized binding in source order. A later comma declarator is not visible early. The provisional binding retains source storage and type facts, so self-reference already uses the final index and taking the address of a pending `register` object remains invalid. The function body's outer compound shares the parameter scope: redeclaring a parameter there is rejected, while a nested block may shadow it. Lookup follows C's ordinary-identifier scopes from innermost block binding to definition parameter to file binding; leaving a compound restores the hidden outer local, parameter, file object, typedef, or enumerator. Body expressions include file bindings, definition parameters, block bindings, concatenated ordinary narrow string literals, target-typed integer and ordinary narrow character constants, parentheses, fixed-argument prototyped calls, explicit scalar/void casts, unary `+`, `-`, `~`, `!`, address-of, and dereference, prefix/postfix increment/decrement, direct and anonymously promoted `.`/`->` member designators, subscripting, multiplicative/additive/shift/relational/equality/bitwise/logical binary operators, right-associative simple or compound assignment, and matching-record conditional values. Integer and character escapes use source bytes plus simple/octal/hexadecimal forms; an ordinary character byte with its high bit set is sign-extended to target signed `int` under Cupid i386's signed plain-`char` rule. Decoded strings own one trailing null and receive fixed `char[N]` types. A return statement stores either `CTOOL_C_AST_NONE` for valid `return;` from `void` or a converted scalar or compatible aggregate expression for a non-void result. An `IF` statement stores a required scalar condition after the ordinary lvalue, array, or function conversion, one required body, and an optional `else_body`; an absent alternative uses `CTOOL_C_AST_NONE`, both bodies precede their parent, and recursive statement parsing gives a dangling `else` to the nearest unmatched `if`. An `if` adds no scope or jump target. A `switch` adds a breakable target while its body is parsed but no loop or implicit scope. Calls enforce prototype arity and accept compatible scalar or aggregate arguments plus integer and pointer-qualification assignment conversion. A variadic prototype may be called with exactly its named arguments, but an argument in the ellipsis fails closed until integer and floating default argument promotions have explicit AST representation. Valid decimal and hexadecimal floating constants are recognized as such and receive an explicit unsupported-expression diagnostic rather than falling through to a malformed-integer diagnostic. A floating source or target at the shared assignment-conversion seam and a floating operand to `!`, `&&`, or `||` receive distinct unsupported diagnostics until floating arithmetic, numeric conversion, and ABI lowering exist.

The first counted-control layer represents C11 `for` without inventing a control-flow graph prematurely. `CTOOL_C_STATEMENT_FOR` retains initializer, condition, iteration, and body fields. The initializer field is `CTOOL_C_AST_NONE` when omitted; when present, it names either the existing non-null expression-statement form or the existing declaration-statement form. The condition and iteration are optional, and the body is required. A missing condition likewise uses `CTOOL_C_AST_NONE` and denotes C's nonzero constant, while a present condition receives lvalue, array, or function conversion and must be scalar. Integer and pointer conditions are represented now; a floating condition is valid C but receives the precise deferred-feature diagnostic until floating conversions exist, whereas aggregate and `void` conditions are constraint failures. A present iteration expression receives the ordinary default conversion. `BREAK` and `CONTINUE` are targetless postorder leaves; lowering resolves the nearest enclosing switch or loop target, while the frontend rejects either outside its permitted enclosing control context. The initializer and body precede their `FOR` parent, and every expression remains postorder in the existing expression table.

C11's loop scope is explicit. A `for` declaration initializer accepts only complete automatic objects with none, `auto`, or `register` storage under the existing declaration constraints; `typedef`, `extern`, `static`, function declarations/specifiers, and other nonautomatic forms fail at the initializer rather than being silently downgraded. The scope covers the declaration initializer, condition, iteration, and body, permits an initializer to shadow a function parameter, and expires immediately after the loop. Parsing now distinguishes a compound statement's block items from the statement grammar: declarations remain valid block items and declaration initializers, but a declaration cannot serve directly as the loop body without a compound statement. This split also makes standalone and loop-body `;` publish a null expression statement. Comma expressions remain outside the body-expression subset; comma-separated declaration declarators remain supported, but an expression initializer such as `i = 0, i = 1` reaches the explicit expression boundary.

Semantic conversions remain explicit one-child AST nodes rather than facts that IR must reconstruct. Lvalue-to-value, array-to-pointer, function-to-pointer, qualification, integer-promotion, usual-arithmetic, and assignment conversions each publish their result type while preserving the source expression as a child. Lvalue conversion removes top-level `const`, `volatile`, and `_Atomic` from the result while the child retains the qualified object type needed to select the load. Array decay transfers array qualification to the pointed-to element, including qualification inherited from a containing record. Scalar initialization applies that lvalue conversion to the declared object type and then reuses the shared assignment-conversion operation, but does not apply assignment's modifiable-lvalue check because a declaration is not an assignment expression. The binding therefore retains its qualified or atomic object type while its initializer root is the converted, unqualified value expression. Plain assignment retains its raw modifiable-lvalue child, converts its right operand to the left object type, and produces an unqualified result; its `computation_type` equals that result type, an invariant rechecked while freezing the public graph. Compound assignments retain the raw left designator exactly once and record the promoted/usual-arithmetic computation type separately from the unqualified stored/result type. Shift assignments use the promoted left type without inventing a usual-arithmetic conversion, and pointer `+=`/`-=` use the pointer result as their computation type plus a promoted integer right child. Prefix/postfix updates likewise publish one raw designator child, distinct operations, the stored/result type, and the promoted integer or pointer computation type. Compatible enum/integer assignment still publishes an explicit assignment conversion to the enum destination instead of disappearing through the compatibility relation. Integer operators use explicit Cupid i386 ranks `_Bool < char < short < int < long < long long`, with signed plain `char`, 32-bit `int` and `long`, and 64-bit `long long`. Integer promotions and the usual arithmetic conversions are represented in the AST, so the ILP32 case `long + unsigned int` resolves to `unsigned long` instead of inheriting a host-compiler rank or width. A narrow unsigned-`int` bit-field whose values fit target signed `int` publishes lvalue conversion to unsigned `int` followed by integer promotion to signed `int`; a 32-bit field remains unsigned. Comparisons and logical operators produce target signed `int`. The callee and converted arguments are the ordered call children, and the call type is the function result.

`CTOOL_C_EXPRESSION_CAST` retains the explicitly requested scalar or `void` destination type instead of disguising source casts as implicit conversions. Its child still undergoes the ordinary lvalue, array, or function conversion, including for a `void` discard, so volatile accesses and decay cannot disappear. Address-of and dereference are unary nodes with their source-semantic child, so later lowering can distinguish an address computation from a load. Addressability tracks the C exclusions for bit-fields and `register` objects through every member hop, while member lvalues inherit the effective record and member qualification. Every `CTOOL_C_EXPRESSION_MEMBER` names one direct member index in the ADR 0013 graph. Lookup through an anonymous structure or union resolves a promoted name to an ordered path and publishes one member node per direct hop; no synthetic flattened member or pre-summed offset is introduced into the public AST.

Pointer addition accepts pointer-plus-integer in either source order, pointer subtraction accepts an integer right operand, and compatible pointer subtraction produces target signed `int`, the Cupid i386 `ptrdiff_t` representation. Every participating pointer must refer to a complete object type; `void`, function, and incomplete referents fail precisely. Pointer difference ignores pointed-to `const`, `volatile`, and `restrict` while retaining atomic type identity. A subscript is normalized to one pointer-addition node followed by dereference, matching C's `*((a) + (b))` definition while retaining operand order, target integer promotion, element qualification, lvalue state, and one lowering path. Pointer comparison and destination-typed null-pointer conversion are represented.

Ordinary body expressions are type-checked but deliberately not folded into public constant nodes. Source such as `1 / 0`, `2147483647 + 1`, or `1 << 32` remains a typed runtime expression rather than being folded away. Runtime values do carry private constant-expression form and target-value metadata so pointer contexts can distinguish exact C11 null pointer constants. That metadata does not alter or replace the public AST. The deliberate public-folding exception is a non-VLA layout query: `sizeof`, alignment queries, and `__builtin_offsetof` become target unsigned 32-bit integer constants after their operands or member paths have been checked against the current Cupid i386 graph. GNU expression alignment uses the selected final member declaration; neither a packed anonymous-promotion container, a named packed container, nor an aligned containing object changes that member's reported alignment. This is distinct from enumerator, bound, bit-field, and `_Static_assert` expressions, where the existing evaluator must diagnose selected arithmetic faults and suppress only faults in a skipped short-circuit operand. Keeping the policies separate preserves runtime source while still sharing target types, conversions, member lookup, and layout.

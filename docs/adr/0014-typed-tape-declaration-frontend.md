# Build CupidC declarations directly from the typed preprocessing tape

CupidC declarations will live behind one freestanding `ctool_c_parse` operation. The operation borrows ADR 0012's immutable preprocessing-token tape plus an explicit C/Cupid language-mode request and publishes one immutable, job-owned translation-unit result. That result contains the ADR 0013 indexed type/member/parameter graph, the corresponding completed i386 layouts, canonical ordinary bindings, named tags, parameter metadata, source-ordered block bindings, and the initial immutable typed function-body AST tables. File bindings distinguish typedefs, objects, functions, and enumerators; the current block-binding slice publishes complete automatic objects. A canonical file binding retains the first declaration's storage spelling and dual location while its linkage and type describe the merged entity; a composite type is symbol-local and never mutates a shared typedef graph. Enumerator bindings retain their target-width integer bit pattern and evaluated signedness. Every public name and both its presumed and immutable physical source location are copied into the job arena; the caller's tape and request are borrowed only for the call.

This is a typed-tape frontend, not a text adapter around the private kernel compiler. It consumes the typed tape as the only lexical authority, including already-expanded spellings, source provenance, and per-token `#pragma pack` caps. A successful call invokes type layout before publishing the translation unit. Any parser, semantic, storage, limit, or layout failure zeros the result, preserves a structured diagnostic, and rewinds every allocation made since the operation mark. The input tape is never mutated. A same-job recovery contract parses a valid declaration immediately after a precise syntax failure without invalidating the earlier preprocessing result. A separate ownership contract builds a tape in caller storage, parses it, destroys and overwrites every borrowed token spelling and path, and then rereads file and block bindings, tags, members, parameters, definitions, statements, and expression names/locations plus decoded literal bytes from the published unit.

The first active-source proof parses the unchanged `/kernel/fs/fat16.h` closure under the exact checked `KERNEL_I386` preprocessing profile. The authoritative tape has 710 tokens: 169 from `/kernel/core/types.h`, 129 from `/kernel/fs/blockdev.h`, and 412 from `/kernel/fs/fat16.h`. It yields 45 external declarations (24 typedefs and 21 functions), 50 ordinary bindings including five enumerators, 12 anonymous records with 78 members, two anonymous enums with five enumerators, and 27 function types with 44 normalized parameters, of which three are abstract. No tag name is invented for an anonymous definition. The frontend-selected compatible type for both nonnegative active enums is unsigned `int`; source `bool` remains an enum typedef rather than being collapsed into `_Bool`.

The next active-source gate keeps four exact unchanged header closures under that same profile. `/kernel/core/kernel.h` is 569 tokens/69 canonical bindings and retains external `print` at its first declaration in `/kernel/cpu/isr.h:7`; `/kernel/cpu/irq.h` is 512/57 and retains the no-linkage `irq_handler_t` typedef from `isr.h:18`; `/kernel/lang/cupidscript.h` is 1,867/177 and retains `script_context_t` from `cupidscript_streams.h:13`; `/kernel/lang/shell.h` is 591/74 and retains its one external `shell_jit_program_is_running` binding at line 81. A Python contract derives the exact 152-header active non-Doom cohort from the checked audit and invokes the public header-sweep mode. Compatible redeclarations first moved the gate to 140 successes; semantic `packed`, `aligned`, and `noreturn` support moved it to 146. File-scope static assertions now parse the unchanged `/kernel/smp/percpu.h:36` check and prove `per_cpu_t` remains `128/64`. The gate remains at 147: unchanged `debug.h` publishes both static-inline call-only bodies, and all five remaining roots now consume their first local declaration before stopping precisely on GNU inline assembly. The three SMP roots reach `percpu.h:44`; `ports.h` reaches line 8 and `cpu.h` reaches line 23. All five failures are therefore explicit extended-assembly boundaries rather than declaration, function-specifier, or brace failures. The [active-source audit](../bootstrap/ACTIVE-SOURCE-AUDIT.md) pins 107 `inline` occurrences across 20 files. This is frontend compatibility evidence, not production ownership.

The source-driven layouts reproduce the existing manual ABI oracles. `key_event_t`, `keyboard_buffer_t`, `keyboard_state_t`, `timer_state_t`, `timer_measure_t`, and `block_device_t` are respectively `12/4`, `3076/4`, `5156/4`, `20/4`, `16/4`, and `24/4`. The packed FAT partition, MBR, boot-sector, and directory-entry records are `16/1`, `512/1`, `36/1`, and `32/1`; the runtime filesystem and file records are `36/4` and `24/4`. All 54 FAT member offsets are pinned, and exactly 36 FAT wire members carry pack cap one while the 42 natural members carry no cap. Replacing only the already-expanded token for `KEYBOARD_BUFFER_SIZE` from `256` with `2` produces `keyboard_buffer_t` at `28/4` and `keyboard_state_t` at `2108/4`. That mutation proves the parser neither reopens source files nor reconstructs the old manual graph.

The frontend owns the declaration-specifier subset contracted by this slice, declarator precedence, pointers, fixed and unspecified arrays, function prototypes, parameter array/function adjustment, records, unions, Cupid classes, enums, typedef lookup, the represented storage classes and qualifiers, bit fields, and their declaration-time language rules. C11 `inline` is accepted only for declarations of function identifiers, repeated use is idempotent, non-function placements fail transactionally at the specifier, and compatible declarations OR one `CTOOL_C_FUNCTION_DECL_INLINE` summary bit into the canonical binding without changing its established C linkage. That summary is deliberately not definition policy: each function-definition record now retains its own declared type, `inline`, and storage spelling before external-inline finalization or emission. `_Thread_local`, `_Noreturn`, `_Alignas`, `_Atomic(type-name)`, and complex/imaginary type specifiers remain explicit pending syntax rather than being implied by that ownership claim. Its namespace contract keeps ordinary identifiers and tags separate, gives function prototypes their required tag/enumerator scope, rejects duplicate parameter names, and does not leak prototype-scope definitions into file scope. It rejects reserved C words in identifier positions and additionally reserves Cupid words only in Cupid mode, so `class` remains an ordinary identifier in C mode. Published file-scope binding metadata distinguishes no storage class, typedef, extern, and static plus none/internal/external linkage. Visible `static` followed by `extern`, and a static function followed by an unadorned declaration, retain internal linkage; an external declaration followed by `static`, or a static object followed by an unadorned external-linkage declaration, conflicts. File-scope auto/register are rejected; valid prototype/definition parameter `register` syntax is checked and retained in `ctool_c_parameter_t`.

Compatible object and function declarations use one iterative, memoized type-relation operation. It collapses duplicate/equivalent qualification wrappers, keeps referent qualifiers distinct, pushes array-typedef qualification to the element type, removes top-level parameter `const`/`volatile`/`restrict` while preserving atomic type identity, and treats a same-scope typedef redefinition more strictly than an object/function compatibility check. Old-style function declarations compose with prototypes only when every prototype parameter survives default argument promotion and no ellipsis is present. Incomplete and fixed arrays compose to the fixed form; opposing refinements in separate function parameters create a new composite parameter slice. Canonical binding replacement is copy-on-write: `typedef int A[]; extern A x; extern int x[4]; extern A y;` completes only `x`, leaving `A` and `y` incomplete. Repeated-pair work is memoized and stored in checked buffers, so a 24-level two-parameter compatibility DAG stays bounded, while 512 iteratively parsed pointer levels prove the 256 source-syntax limit is not misapplied to derived type graphs.

The GNU declaration subset parses normalized `__attribute` and `__attribute__` lists at active record, member, typedef, object/function, prefix, and interposed placements. `packed` updates the record/member layout policy; record alignment raises record layout; member alignment raises member placement; a typedef alignment creates an exact `CTOOL_C_TYPE_ALIGNED` wrapper without mutating its record; and object/function alignment publishes a minimum entity alignment on the canonical binding. `noreturn` is a binding-level function effect, not a type-compatibility distinction, and compatible declarations merge it in either order. Aligned wrappers compare transparently as C types while asymmetric composite selection retains the strongest alignment recursively and repeated typedefs retain the maximum. When alignment and structural refinements occur at opposing wrapper depths, composition separates common outer qualifiers from the structural children, combines the children, and reapplies exact alignment and qualifiers in source-semantic order. Contracts pin incomplete-aligned/fixed-array composition, nested exact wrappers, top-level `const`, and `_Atomic` minimum alignment in both declaration orders. Default-promotion checks unwrap alignment, so an aligned `char` prototype still conflicts with an old-style declaration in either order. Cupid i386 assigns bare `aligned`/`aligned()` the target default 16 and deterministically takes the maximum of repeated alignment requests. Unknown attributes, disabled GNU mode, malformed groups, invalid alignment, wrong arity, wrong targets, and currently unrepresented parameter attributes receive transactional `CTOOL_C_PARSE_DIAG_ATTRIBUTE` failures rather than being skipped. Unchanged source contracts pin `process_t` at `656/16` with `fp_state` at byte 80, the two packed IDT records, all five packed loader records, the per-CPU body at `128/4` with its typedef at `128/64`, both packed/aligned E1000 descriptors, and the three panic functions' retained `noreturn` effect.

Declaration-time completeness and qualification are checked where C requires them rather than inferred from final graph completeness. Every syntactic array declarator requires a complete object element type when the array is constructed, before a typedef can publish it or a parameter array can adjust to a pointer; completing a referenced tag later cannot repair an invalid array. In strict C mode an array element also cannot recursively contain a flexible-array member. By-value record members have the same temporal completeness rule, while an incomplete type remains valid in a function prototype and an external array with a complete element and unspecified bound retains its incomplete array representation. Void objects, arrays, and members are rejected; the single unnamed `void` prototype parameter is normalized to an empty prototype; named or non-sole void parameters are rejected. Array parameters adjust to pointers while retaining element qualification, and abstract `int ()`/`int (void)` parameter types adjust to pointer-to-function without manufacturing names. Qualified array objects retain their array representation, and declarations that would return arrays or functions are rejected. Flexible-array eligibility counts both direct and anonymously promoted names, accepts a promoted-name-plus-final-flexible-array record, rejects a sole flexible member, rejects an anonymous enum as a member, diagnoses promoted-name collisions, and propagates the strict-C embedding restriction through containing unions without rejecting the permitted union construction itself.

The operation now publishes the first body layer under ADR 0003 without adding a second parse seam. `ctool_c_translation_unit_t` carries immutable flat function-definition, block-binding, statement, expression, statement-child, and expression-child tables beside the declaration graph. Function definitions name their canonical binding but separately retain the exact definition declarator type, source storage, `inline` flags, body, and dual location. Definition parameter expressions index the definition type's existing parameter slice, and `ctool_c_parameter_t` now retains `register`/none storage. A declaration statement indexes its ordered slice of `ctool_c_block_binding_t`; each binding owns its object name, source storage spelling, semantic type, and dual location. A block-binding expression indexes that table directly. Statement and expression nodes are postorder: every child index is smaller than its parent, slices preserve source order, and inapplicable indices use `CTOOL_C_AST_NONE`. Closed lexical scopes are parser scratch rather than public state, so their source-ordered bindings survive for later IR without exposing scope stacks, token pointers, or recursive ownership.

The initial body grammar deliberately follows unchanged active cohorts. It accepts nested compound and expression statements plus uninitialized complete automatic-object declarations with none, `auto`, or `register` storage, comma-separated declarators, pointers, and types already expressible through the supported declaration subset. The function body's outer compound shares the parameter scope: redeclaring a parameter there is rejected, while a nested block may shadow it. Lookup follows C's ordinary-identifier scopes from innermost block binding to definition parameter to file binding; leaving a compound restores the hidden outer local, parameter, file object, or typedef. Expressions otherwise remain file bindings, definition parameters, block bindings, concatenated ordinary narrow string literals using source bytes plus simple/octal/hexadecimal escapes, and fixed-argument prototyped calls. Ordinary strings are decoded to owned Cupid target bytes with one trailing null and receive fixed `char[N]` types in the shared graph. Calls enforce prototype arity and currently accept compatible arguments plus pointer qualification addition. A variadic prototype may be called with exactly its named arguments, but an argument in the ellipsis fails closed until integer and floating default argument promotions have explicit AST representation. Semantic conversions are explicit one-child AST nodes rather than facts that IR must reconstruct: lvalue-to-value, array-to-pointer, function-to-pointer, and qualification conversions each publish their result type while preserving the source expression as a child. Lvalue conversion removes top-level `const`, `volatile`, and `_Atomic` from the result while the child retains the qualified object type needed to select the load. The callee and converted arguments are the ordered call children, and the call type is the function result.

A synthetic public contract separates canonical from definition-local facts with an earlier `static inline` prototype followed by a non-inline definition whose second parameter is `register`. It proves the named-only variadic call boundary, parameter-before-file-binding resolution, concatenated literals decoded to `"\n"` bytes, a nested compound, conversion result types, postorder slices, exact diagnostics, tape immutability, arena rollback, same-job recovery, and survival of earlier AST results. A second positive oracle converts a `const volatile _Atomic` file object and proves the conversion result is the sink's unqualified, non-atomic parameter type while the source child retains all three qualifiers. Eight exact call/body negatives cover too few and too many arguments, an extra variadic argument, an incompatible argument, an undeclared identifier, duplicate definitions, a definition hidden in a declarator list, and an oversized narrow-string escape. A generated 256-deep call expression additionally proves host recursion stops transactionally at the shared syntax limit. The block-binding oracle publishes nine objects from seven declaration statements across two functions and checks mixed declarators, none/`auto`/`register` storage, nested shadowing of a parameter, typedef, and outer local, file-object shadowing, scope restoration, and source-ordered references. Fifteen exact negative cases cover same-scope duplicates, an outer-body parameter collision, initializers, unsupported storage classes, block typedefs/functions/tag specifiers/static assertions/attributes/GNU assembly, incomplete or void objects, and use after scope exit. Every failure revalidates the earlier successful unit. A borrowed-tape contract destroys every token spelling and path after parsing and then rereads a block binding and its declaration slice in addition to the existing definition/statement/expression dual locations and decoded literal bytes. The unchanged 629-token `/kernel/core/debug.h` closure still publishes two static-inline definitions, eight statements, 32 expressions, 14 explicit conversions, and two owned newline literals. The active non-Doom header frontier remains 147/152, but all five remaining failures move through the first local declaration to explicit GNU inline-assembly diagnostics at `ports.h:8`, `cpu.h:23`, and `percpu.h:44` through the three SMP roots.

Brace skipping was rejected because it would report false source coverage and discard exactly the definition-local and typed facts needed by IR. Reusing the canonical binding's storage or OR-summary was rejected because compatible earlier declarations can differ from the definition. Call-edge-only conversion metadata was rejected because returns, assignments, initializers, and operators need the same conversion vocabulary; implicit conversions are ordinary expression nodes instead. Recursive public AST pointers and exposed scope tables were rejected in favor of the same immutable indexed ownership model as the type graph.
Integer constant expressions currently implement the declaration subset required for enumerators, array bounds, bit-field widths, and static assertions: target-typed integer literals and enumerators, parentheses, unary sign/complement/logical-not, multiplicative, additive, shift, all six relational/equality operators, bitwise operators, and logical `&&`/`||`. Evaluation uses explicit Cupid i386 integer ranks, widths, usual arithmetic conversions, signed overflow rules, unsigned wrapping, truncating signed division/remainder, arithmetic signed right shift, checked shift counts, and signed relational comparison rather than host `uint64_t` behavior. Comparisons and logical operators produce target signed `int` zero or one. Logical operands retain C short-circuit evaluation: a skipped operand is still parsed, resolves identifiers, and enforces type/operator constraints, while divide/remainder by zero, signed arithmetic or unary-negation overflow, and invalid or overflowing shifts are suppressed only when evaluation is skipped. Contracts drive the shared grammar through enum initializers and assertions plus the existing array-bound and bit-field-width seams; cover precedence, mixed signed/unsigned comparisons and values, signed-`int` and signed-`long long` compatible enum selection, invalid suffixes and literals, selected versus skipped arithmetic faults, invalid skipped operands, and implicit-enumerator overflow. Conditional, comma, cast, character, and address expressions are not yet part of this grammar and fail closed.

`sizeof(type-name)` is a target operation within that grammar, not a host `sizeof`. It accepts declaration specifiers and abstract pointer, array, and function-pointer declarators, returns Cupid i386 `size_t` as unsigned 32-bit, and reuses ADR 0013 for scalar, pointer, array, qualifier, alignment, and record representation. Each query snapshots the current logical type versions plus already completed member and parameter slices, lays out that graph in nested job-arena scratch, copies the requested result, and rewinds the snapshot. This preserves declaration-order completeness: `void`, functions, incomplete records/arrays, and a record inspected inside its own definition fail, while a pointer to an incomplete record succeeds and later completion cannot repair an earlier invalid query. Type-name nodes constructed while checking an assertion remain part of the published semantic graph; the assertion itself publishes no binding or record member. `sizeof unary-expression` and parenthesized expression operands require the future typed unevaluated-expression frontend and receive an explicit unsupported diagnostic rather than being approximated by the integer evaluator.

C11 `_Static_assert` declarations are accepted at file and record scope, require an integer constant expression plus one or more adjacent string-literal tokens, accept every nonzero value, and emit a dedicated diagnostic containing the concatenated source message when false. A record assertion contributes no member and cannot alter layout. Diagnostic construction is bounded by `diagnostic_message_bytes` independently of `output_bytes`; an unrepresentable message is a limit failure rather than a false-success diagnostic without the required text. Assertions are validation-only and are not stored in `ctool_c_translation_unit_t`. Block placement remains an explicit unsupported statement until that assertion form is added to the body grammar.

Declaration, member, tag, parameter, compatibility, and composite-type storage uses checked job buffers rather than the private frontend's fixed tables. A synthetic translation unit with 256 typedefs and one 256-member record proves the old 16-typedef and 32-member ceilings are gone. Namespace and member lookup is currently linear; this is practical for the contracted flat scale but must become indexed while preserving the contracted scope ordering before the complete kernel, Doom, and hosted-tool corpus can make lookup cost predictable. Recursive flexible-array containment uses visited logical-type IDs plus a checked `types + members + 1` work-item budget. A 24-level two-edge shared DAG without a flexible leaf must parse successfully, while the equivalent flexible-leaf graph must be rejected; removing visited-node suppression makes the bounded positive case fail and is therefore caught mechanically. Recursively nested declarators, constant-expression parentheses, and records have one public limit of 256 levels. This exceeds the C11 minimum translation limits and deliberately converts pathological 3,000-level source syntax into `CTOOL_C_PARSE_DIAG_LIMIT` with full rollback instead of consuming the host stack. Type compatibility/composition is iterative and consumes checked storage instead of that syntax limit.

The public boundary is fail-closed. Null tape/request pointers, an invalid mode or Boolean, a nonzero tape with no token array, and invalid token pack metadata produce the exact invalid-request diagnostic without consuming input or arena state. A one-byte configured output limit produces `CTOOL_C_PARSE_DIAG_LIMIT`, while a second limit is set to the exact pre-composite type-vector size for two compatible opposing function prototypes: parsing reaches the composite declaration, fails its next type append as a limit rather than an internal error, zeros output, restores the arena, preserves the tape, and leaves an earlier unit valid. A third measured limit reaches a trailing aligned typedef and fails exactly when its wrapper would extend the type vector. Static-assert contracts additionally prove that false-message construction succeeds with `output_bytes` below the independent diagnostic limit, that an overlong message returns the public limit diagnostic, and that a 96-pointer current graph exhausts a 4 KiB arena during the nested `sizeof` layout snapshot. Declarator, record, constant-expression, and now nested call-expression recursion share the 256-level public syntax limit; the call-depth case proves rollback without exhausting the host stack. Every case zeros the failed result, restores the exact operation mark, preserves its tape, and leaves an earlier unit valid. A syntactically valid zero-bound array separately proves that a downstream ADR 0013 layout error rolls back the entire outer parse operation. Object initializers, unsupported body statements/expressions, and Cupid `#exe` markers each return a documented fail-closed diagnostic rather than being ignored.

Adapting the private kernel parser was rejected because it re-lexes flattened text, loses counted spellings, columns, physical/presumed provenance, pack metadata, qualifiers, and stable type identity, silently truncates several fixed-size tables, and emits machine code while parsing. The first success-only FAT contract was also rejected as sufficient evidence: adversarial review exposed gaps in prototype scope, reserved identifiers, promoted flexible-array names, temporal completeness, qualifier/storage/type legality, abstract parameters, integer semantics, enum compatibility, and pathological nesting. Those became positive and transactional negative contracts. Final semantic review then found null-token-array pointer arithmetic, enumerator reuse based on initializer signedness instead of declared type, incorrect outer/prototype shadowing, rejected valid incomplete prototypes and external arrays, and incomplete recursive flexible-array propagation. Standards review additionally found exponential traversal over a shared flexible-array type DAG, incorrect union treatment, nested prototype lookup ordering, a Windows-only audit expectation, and position-dependent handling of typed Cupid `#exe` markers. The fixes use declared enumerator types, ordered prototype marks, visited type traversal with a checked work bound, host-suffix-aware audit assertions, and one unsupported-marker normalization at the parser diagnostic seam. A last construction-time regression proved that typedef, adjusted-parameter, and union-member arrays could still bypass incomplete/flexible element rules; four red counterexamples moved the constraint into array construction. The first redeclaration implementation was likewise rejected after review: it recursively compared types under the source-syntax limit, accepted merely compatible typedef redefinitions, rejected valid parameter/array qualifier spellings and old-style prototypes, revisited a shared type DAG exponentially, and completed shared typedef nodes in place. Iterative memoized relations, strict same-type mode, normalized parameter types, and symbol-local composite construction replace that design. Raw unsigned-64 arithmetic for every integer expression was rejected after it produced incorrect signed division, conversion, overflow, and enum-selection behavior. Unbounded source-syntax recursion was replaced by the deterministic public limit. Static analysis findings are fixed rather than suppressed.

This operation does not yet parse initialized, static, external, typedef, function, direct-tag, or assertion declarations at block scope; return/control statements; GNU extended assembly; general operators; calls without prototypes; extra variadic arguments/default promotions; universal-character or non-ordinary string literals; or file/object initializers. It does not yet publish/finalize tentative entity or external-inline state; accept the remaining active attributes (`section`, `weak`, `used`, `noinline`, `unused`, `naked`, and `target`), `_Thread_local`, `_Noreturn`, `_Alignas`, `_Atomic(type-name)`, or complex/imaginary type specifiers; implement the remaining constant-expression or typed unevaluated-expression grammar; represent Cupid `#exe` policy or other Cupid extensions beyond declaration keywords/classes; complete the statement/expression typed AST; lower it to IR; generate machine code; or emit objects. Unsupported syntax is diagnosed rather than skipped; with GNU extensions enabled, `asm`, `__asm`, and `__asm__` at statement position receive an explicit statement diagnostic, and tentative incomplete-array declarations remain in their parsed form because object tentative/definition state is not represented yet. Of the 22 active static assertions in four files, the public contracts now prove the `percpu.h` type-name assertion and all three executable-arena alignment assertions from an unchanged prefix of `exec.c`. The other 18 still require `sizeof(expression)`, `__alignof__`, casts/member access, or `__builtin_offsetof`. The request's mode and GNU-extension setting must currently match those used to produce the tape because preprocessing provenance does not yet carry that configuration.

This capability is hosted bootstrap evidence only. GCC or Clang still builds the frontend and its contract. The private in-kernel CupidC frontend and code generator are unchanged, no production OS artifact consumes the new translation unit, and no boot or runtime behavior is claimed. GNU extended assembly is now the exact next unchanged-header frontier at `ports.h:8`, `cpu.h:23`, and `percpu.h:44`; the return/control/member/arithmetic forms after it remain body work. Definition-local `inline` intent is retained, while external-inline classification remains translation-unit finalization work; completing the `.c` assertion cohort also needs typed unevaluated expressions and offset/alignment builtins. The remaining active GNU attributes and extended assembly remain required for that cohort. Only a complete typed AST, linear IR, and object lowering can let the kernel adapter replace its private preprocessing/parser/code-generation pipeline. `TempleOS/` remains excluded reference material.

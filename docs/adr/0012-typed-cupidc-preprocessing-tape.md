# Keep CupidC preprocessing behind one typed token-tape operation

CupidC preprocessing will live in one freestanding `ctool_c_preprocess` operation above the job-owned Toolchain core. Its request type reserves the final ordered contract: primary source, language mode, include roots, forced includes, and macro actions. Macro actions execute in request order before forced includes, and forced includes execute before the primary source. A successful call returns one immutable, source-located preprocessing-token tape; all token spellings and logical paths are owned by the job arena. A failed call publishes a structured diagnostic, rewinds every operation allocation, and leaves no partial result. Hosted and kernel callers use the existing file-store, arena, path, and diagnostic adapters rather than introducing another filesystem port.

The tape is the stable seam. Include traversal and caching, macro tables and hide sets, conditional stacks, pragma state, expansion traces, spelling construction, and limits remain private. Each public token carries its effective `#pragma pack` alignment so a parser does not need to replay pragma history. Active Cupid `#exe` is represented as a typed language marker; preprocessing does not learn JIT, AOT, shell, or executable-memory policy. Target facts such as pointer width, byte order, and platform macros arrive as explicit ordered macro actions so they describe the compilation target instead of the host running CupidC. Language-owned predefined macros remain the preprocessor's responsibility.

A larger profile/renderer/view interface was rejected because no second caller requires public include traces, dependency graphs, or textual `-E` output. Returning flattened text plus a source map was rejected because every compiler consumer would re-lex it, pack state would need a parallel event stream, and Cupid execution policy would leak into profiles. The typed tape makes macro expansion and source provenance authoritative once while leaving room for a renderer later if a real caller establishes that need.

The initial implementation normalizes escaped newlines into one arena-owned source image and records only the output offsets of removed newlines. This preserves physical token locations and contiguous spellings even when a token crosses a splice, without a per-character origin map. Private lexer state distinguishes logical-line start and leading space: comments introduce space, retained newlines reset line start, and removed newlines do neither. The directive layer now owns ordered object and standard variadic function macros, sequence rescanning with immutable hide sets, raw stringification, token paste, file-local conditional stacks, parent/root include search, forced inputs, and a cache of immutable raw token tapes. Function definitions compile replacement-token parameter indices and a needs-expansion bitmap once, keeping each invocation linear in its parameters, replacement, and substituted output instead of repeatedly searching parameter spellings. Variadic arguments retain both a raw sequence and syntactic omission state; paste and stringify consume raw tokens while ordinary occurrences use one pre-expanded view.

Paste operators and placemarkers are private work-node kinds and can never reach the public tape. Operand spellings are concatenated and re-lexed as exactly one complete preprocessing token; invalid results fail transactionally. Pasted-token hide sets use the Prosser intersection rule, while placemarker survival preserves the real token's hide set. This deliberately keeps a coherent documented algorithm for underspecified mixed-hide abuse even where modern GCC/Clang happen to choose another result. A pasted `##` is an ordinary result token rather than a second operator pass. Only the exact definition-owned comma + paste + synthetic variadic slot pattern receives GNU comma deletion, and only when the pack was syntactically omitted. Ordinary omitted substitution follows GCC and leaves surrounding punctuation; explicit empty/comment-empty/macro-empty packs remain distinct. Named GNU `args...` syntax stays an explicit unsupported boundary because no active source requires it.

Token separation is private worklist state as well as token state. If an ordinary empty argument, an empty macro, or a surviving placemarker removes the only token carrying a required separator, that separator moves to the next material token or to trailing replacement state for the caller suffix. The GNU comma/paste/variable-slot extension is deliberately different: deleting the special triple also deletes spacing attached to that punctuation, and a present raw variable operand does not inherit whitespace around `##`. These rules preserve host-equivalent nested stringification without manufacturing whitespace that the paste operator removed.

Conditional expressions use a private bounded evaluator after macro expansion and return no parser or arithmetic state through the public seam. C signed and unsigned integer types are evaluated as 64-bit two's-complement `intmax_t`/`uintmax_t`; signed overflow is diagnosed and unsigned arithmetic wraps. Negative signed right shift is arithmetic. `&&`, `||`, and `?:` still parse skipped operands but suppress arithmetic faults there. A comma expression is accepted only inside such an unevaluated subexpression, as C permits; a selected/evaluated comma is diagnosed. Ordinary characters use signed 8-bit values, ordinary multicharacter constants pack at most four bytes from left to right, `wchar_t` is signed 32-bit, and `char16_t`/`char32_t` are unsigned 16/32-bit values. Universal-character names enforce the C scalar and low-code-point restrictions, while numeric escapes use the destination character type's full range. These are Cupid target rules, never facts inferred from the compiler hosting the bootstrap. Binary literals, `\e`, and oversized decimal promotion are available only when GNU extensions are requested.

A source-written `defined` operator carries private provenance until the outer conditional expansion consumes it. An unused macro argument may discard malformed `defined` text without interpretation. Substitution or paste clears that provenance, so a `defined` operator produced by macro expansion is rejected deterministically instead of relying on C's undefined behavior. The identifier is reserved against source and configured macro definition or undefinition, but remains legal as a function-macro parameter.

The preprocessor owns all seven C11 language predefined macros. `__FILE__` and `__LINE__` follow invocation locations; file spellings escape quotes, backslashes, and non-printable path bytes without retaining caller storage. `__DATE__` and `__TIME__` never read a host clock: callers may provide validated `Mmm dd yyyy` and `hh:mm:ss` translation values, while an omitted seed uses the reproducible bootstrap epoch `Jan  1 1970` / `00:00:00`. Target facts such as pointer width, byte order, ISA, and platform remain ordered macro actions.

Cache membership never implies once-only processing: repeated guardless includes emit repeatedly, while shared macro state terminates ordinary guards and no active-path cycle rule blocks legal recursion. The primary source is seeded into the cache so a caller-supplied source can include itself without requiring a second file-store copy. Active `#pragma once` adds private entered/once state to that cache entry. Identity is the canonical, case-sensitive logical path, so alternate include spellings resolving to one path share the mark while distinct paths with identical bytes remain distinct. Marking a source has no effect outside the current preprocessing operation, and a once directive in an inactive group has no effect.

Active `#pragma pack` state belongs to the translation unit, not an individual source. It crosses forced and ordinary include boundaries and annotates every emitted direct, argument, replacement, predefined, stringified, or pasted token with the effective cap at its invocation position. Immutable cached raw tapes remain unstamped so replaying a guardless header under another ambient cap produces the correct metadata. The private stack supports `pack()`, `pack(n)`, `pack(push)`, `pack(push,n)`, named push with or without a new cap, pop, and nearest-name pop. Accepted integer constants represent 0, 1, 2, 4, 8, or 16 in decimal, octal, or hexadecimal form with standard integer suffixes. Operands deliberately remain raw: GCC 15 rejects a defined macro operand while Clang 22 expands it, so Cupid chooses the deterministic GCC-compatible policy until an active source requires a broader contract. Named pop removes the matched frame and every newer frame. Live depth is bounded at 256, and popped frames are recycled so storage follows maximum simultaneous depth rather than transition count.

Saving/restoring pack state at include boundaries was rejected because it would break the host semantics used by packed wire-layout headers. Stamping raw cached tokens was rejected because cache replay would retain the first include's ambient state. Silently accepting a missing pop or named frame was rejected in favor of a transactional `CTOOL_C_PP_DIAG_PRAGMA_PACK` error. An unmatched final push remains accepted: GCC 15 accepts it while Clang 22 diagnoses it only through a warning category, and the broader behavior avoids making valid host source artificially inexpressible. Unknown pragmas remain explicit unsupported directives. The active-source contract currently classifies exactly one `once` and two balanced `pack(push, 1)`/`pack(pop)` pairs at depth one; both FAT16 headers use those pairs for disk-wire layouts.

An active source-written `#exe` or `%:exe` in Cupid mode requires `{` as its first raw token on the same phase-two logical directive line. A macro cannot manufacture either the directive name or its opener. The output is one `CTOOL_C_PP_TOKEN_CUPID_EXE` token with spelling `exe`, the introducing `#`/`%:` location, and the current pack cap, followed by the ordinary expanded `{` and suffix tokens. Expansion may continue across physical lines until the next directive, so a function-macro call beginning on the directive tail can close on the following line. Later preprocessing directives retain their normal meaning rather than being captured as text. The preprocessor deliberately does not search for a matching brace: closure, nesting, placement, block ASTs, execution order, and JIT/AOT behavior belong to parser/backend policy. Active C11 use is an explicit unsupported-language error, malformed active Cupid use is an input error, inactive use has no effect, and every failure retains the operation's transactional result contract.

The checked active-source contract fails closed on conditional or non-block forms and currently records exactly one ordinary, unconditional block at `bin/feature6_exe.cc:7`. Focused contracts additionally prove digraph recognition, phase-two splicing, cross-line macro expansion, forced/include/once ordering, precise path/line/column/cap ownership, no hidden-function lowering, parser ownership of an unclosed block, and 256 markers without importing the private runtime's 128-block limit.

Source inspection of the private kernel frontend found why its current lowering is not the shared contract. JIT preprocessing rewrites blocks into generated `__cc_exe_N` functions and later discovers them by symbol-prefix scan; AOT skips them with a warning. Its capture limit is 128, brace counting does not model block comments, EOF capture is not diagnosed, capture can cross an include boundary, directives inside a captured block are suppressed, and a single-line AOT block can leave skip state active. The shared tape intentionally corrects the directive suppression and removes hidden names, prefix discovery, capture state, and execution limits from preprocessing. Parser/backend migration must use an explicit ordered representation instead of recreating those leaks. Whether AOT executes or only validates blocks, whether skipped bodies are type-checked, whether blocks are top-level-only or nestable, and whether header-origin markers are language-valid remain later language-policy decisions; preprocessing preserves header markers rather than deciding prematurely.

Macro-expanded includes, presumed locations through `#line`, full active-corpus preprocessing, parser and record-layout consumption of tape metadata, and kernel integration remain required before the tape replaces the private kernel preprocessor. The private kernel frontend currently recognizes once and `#exe` independently and cannot preserve the typed tape's path/column/pack metadata; its record layout also needs real 16-bit size/alignment semantics before the FAT16 ABI can move safely.

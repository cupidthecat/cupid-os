# Keep CupidC preprocessing behind one typed token-tape operation

CupidC preprocessing will live in one freestanding `ctool_c_preprocess` operation above the job-owned Toolchain core. Its request type reserves the final ordered contract: primary source, language mode, include roots, forced includes, and macro actions. Macro actions execute in request order before forced includes, and forced includes execute before the primary source. A successful call returns one immutable, source-located preprocessing-token tape; all token spellings, presumed source names, and canonical physical paths are owned by the job arena. A failed call publishes a structured diagnostic, rewinds every operation allocation, and leaves no partial result. Hosted and kernel callers use the existing file-store, arena, path, and diagnostic adapters rather than introducing another filesystem port.

The tape is the stable seam. Include traversal and caching, macro tables and hide sets, conditional stacks, pragma state, expansion traces, spelling construction, and limits remain private. Each public token carries its effective `#pragma pack` alignment so a parser does not need to replay pragma history. Active Cupid `#exe` is represented as a typed language marker; preprocessing does not learn JIT, AOT, shell, or executable-memory policy. Target facts such as pointer width, byte order, and platform macros arrive as explicit ordered macro actions so they describe the compilation target instead of the host running CupidC. Language-owned predefined macros remain the preprocessor's responsibility.

A larger profile/renderer/view interface was rejected because no second caller requires public include traces, dependency graphs, or textual `-E` output. Returning flattened text plus a source map was rejected because every compiler consumer would re-lex it, pack state would need a parallel event stream, and Cupid execution policy would leak into profiles. The typed tape is the single authority for macro expansion and source provenance. A renderer can be added later if a real caller needs one.

The initial implementation normalizes escaped newlines into one arena-owned source image and records only the output offsets of removed newlines. This preserves physical token locations and contiguous spellings even when a token crosses a splice, without a per-character origin map. Private lexer state distinguishes logical-line start and leading space: comments introduce space, retained newlines reset line start, and removed newlines do neither. The directive layer now owns ordered object and standard variadic function macros, sequence rescanning with immutable hide sets, raw stringification, token paste, file-local conditional stacks, parent/root include search, forced inputs, and a cache of immutable raw token tapes. Function definitions compile replacement-token parameter indices and a needs-expansion bitmap once. Each invocation is therefore linear in its parameters, replacement, and substituted output instead of repeatedly searching parameter spellings. Variadic arguments retain both a raw sequence and syntactic omission state; paste and stringify consume raw tokens while ordinary occurrences use one pre-expanded view.

Paste operators and placemarkers are private work-node kinds and can never reach the public tape. Operand spellings are concatenated and re-lexed as exactly one complete preprocessing token; invalid results fail transactionally. Pasted-token hide sets use the Prosser intersection rule, while placemarker survival preserves the real token's hide set. This deliberately keeps a coherent documented algorithm for underspecified mixed-hide abuse even where modern GCC/Clang happen to choose another result. A pasted `##` is an ordinary result token rather than a second operator pass. Only the exact definition-owned comma + paste + synthetic variadic slot pattern receives GNU comma deletion, and only when the pack was syntactically omitted. Ordinary omitted substitution follows GCC and leaves surrounding punctuation; explicit empty/comment-empty/macro-empty packs remain distinct. Named GNU `args...` syntax stays an explicit unsupported boundary because no active source requires it.

Token separation is private worklist state as well as token state. If an ordinary empty argument, an empty macro, or a surviving placemarker removes the only token carrying a required separator, that separator moves to the next material token or to trailing replacement state for the caller suffix. The GNU comma/paste/variable-slot extension is deliberately different: deleting the special triple also deletes spacing attached to that punctuation, and a present raw variable operand does not inherit whitespace around `##`. These rules preserve host-equivalent nested stringification without manufacturing whitespace that the paste operator removed.

Conditional expressions use a private bounded evaluator after macro expansion and return no parser or arithmetic state through the public seam. C signed and unsigned integer types are evaluated as 64-bit two's-complement `intmax_t`/`uintmax_t`; signed overflow is diagnosed and unsigned arithmetic wraps. Negative signed right shift is arithmetic. `&&`, `||`, and `?:` still parse skipped operands but suppress arithmetic faults there. A comma expression is accepted only inside such an unevaluated subexpression, as C permits; a selected/evaluated comma is diagnosed. Ordinary characters use signed 8-bit values, ordinary multicharacter constants pack at most four bytes from left to right, `wchar_t` is signed 32-bit, and `char16_t`/`char32_t` are unsigned 16/32-bit values. Universal-character names enforce the C scalar and low-code-point restrictions, while numeric escapes use the destination character type's full range. These are Cupid target rules, never facts inferred from the compiler hosting the bootstrap. Binary literals, `\e`, and oversized decimal promotion are available only when GNU extensions are requested.

A source-written `defined` operator carries private provenance until the outer conditional expansion consumes it. An unused macro argument may discard malformed `defined` text without interpretation. Substitution or paste clears that provenance, so a `defined` operator produced by macro expansion is rejected deterministically instead of relying on C's undefined behavior. The identifier is reserved against source and configured macro definition or undefinition, but remains legal as a function-macro parameter.

The preprocessor owns all seven C11 language predefined macros. `__FILE__` and `__LINE__` follow invocation locations; file spellings escape quotes, backslashes, and non-printable path bytes without retaining caller storage. `__DATE__` and `__TIME__` never read a host clock: callers may provide validated `Mmm dd yyyy` and `hh:mm:ss` translation values, while an omitted seed uses the reproducible bootstrap epoch `Jan  1 1970` / `00:00:00`. Target facts such as pointer width, byte order, ISA, and platform remain ordered macro actions.

Active C11 `#line` and `%:line` directives macro-expand their operands, require one decimal digit sequence in the standard 1..2147483647 range, and optionally decode one ordinary narrow string literal into a counted presumed source name. Each public token exposes both that effective presumed name/line and an immutable canonical physical path/line; columns remain physical. Direct and substituted argument tokens retain their physical origins, while replacement-list, stringify, paste, predefined, and Cupid `#exe` output uses the corresponding invocation or directive origin. `__FILE__`, `__LINE__`, downstream structured diagnostics, and unmatched-conditional diagnostics consume presumed locations. Eager whole-source lexical diagnostics remain physical because phase-three tokenization precedes directive execution. An empty presumed name is retained exactly in structured data even though the shared text renderer uses its generic `<toolchain>` label for an empty diagnostic path.

Presumed frames are local to each physical source entry and are replayed from immutable cached raw tapes. Relative include lookup, cache membership, and `#pragma once` identity therefore continue to use canonical physical paths; a child mapping cannot leak into its parent, and returning from a child restores the parent's current mapping. Phase-two splice accounting anchors a directive at the exact physical line following its retained newline, matching GCC's source-line behavior when an operand spans continuations. GNU digit-led line markers remain explicitly unsupported until their flags and include-stack semantics are represented; like other nonconditional directives, they have no effect in an inactive group.

Exact direct include operands remain contextual and opaque: one raw header-name token or one ordinary string token is never macro-expanded. Every other C11 third-form operand is fully rescanned through the same private macro worklist and must finish as exactly one ordinary string token, one surviving contextual header-name token, or a complete `<` ... `>` token sequence with no trailing token. The final form selects the existing search policy, so quotes remain including-parent-first and angles remain eligible-root-only. Adjacent strings, prefixed strings, empty expansions, recursive residues, incomplete angles, and trailing tokens are input errors; a valid missing path remains a not-found error and macro expansion failures retain their own invocation diagnostic.

C leaves expanded angle-token combination implementation-defined. Cupid matches GCC 15 and Clang 22: token spellings concatenate while every surviving `leading_space` boundary contributes exactly one ASCII space, including leading space after `<` and trailing space before `>`. Comments therefore become one separation. A lazily allocated `path_bytes` buffer predates each directive scratch mark. Expansion and reconstruction copy the final spelling into that stable buffer, restore the global expansion-node count, and rewind every temporary node before searching or recursively entering a source. A constrained contract drives a 32-alias operand through a cached forced dispatcher 32,768 times in a 512 KiB arena. Without scratch rewind it exhausts memory, and without count restoration its more than one million expansion nodes hit the global limit; the implemented reuse passes.

The checked active graph currently contains no macro include operand, so this is standards and toolchain compatibility rather than a production-ownership transfer. Its fail-closed closure contract records 658 tracked C-family inputs and exactly 2,343 direct operands: 2,132 quoted and 211 angle. Any future pp-token operand is rejected with source, marker, physical line, normalized operand, and syntactic conditional context until the audit consumes a CupidC dependency trace instead of silently omitting a macro-derived edge. Four present generated C inputs add 12 direct quotes but remain governed by their generator and materialization boundary.

Cache membership never implies once-only processing: repeated guardless includes emit repeatedly, while shared macro state terminates ordinary guards and no active-path cycle rule blocks legal recursion. The primary source is seeded into the cache so a caller-supplied source can include itself without requiring a second file-store copy. Active `#pragma once` adds private entered/once state to that cache entry. Identity is the canonical, case-sensitive logical path, so alternate include spellings resolving to one path share the mark while distinct paths with identical bytes remain distinct. Marking a source has no effect outside the current preprocessing operation, and a once directive in an inactive group has no effect.

Active `#pragma pack` state belongs to the translation unit, not an individual source. It crosses forced and ordinary include boundaries and annotates every emitted direct, argument, replacement, predefined, stringified, or pasted token with the effective cap at its invocation position. Immutable cached raw tapes remain unstamped so replaying a guardless header under another ambient cap produces the correct metadata. The private stack supports `pack()`, `pack(n)`, `pack(push)`, `pack(push,n)`, named push with or without a new cap, pop, and nearest-name pop. Accepted integer constants represent 0, 1, 2, 4, 8, or 16 in decimal, octal, or hexadecimal form with standard integer suffixes. Operands deliberately remain raw: GCC 15 rejects a defined macro operand while Clang 22 expands it, so Cupid chooses the deterministic GCC-compatible policy until an active source requires a broader contract. Named pop removes the matched frame and every newer frame. Live depth is bounded at 256, and popped frames are recycled so storage follows maximum simultaneous depth rather than transition count.

Saving/restoring pack state at include boundaries was rejected because it would break the host semantics used by packed wire-layout headers. Stamping raw cached tokens was rejected because cache replay would retain the first include's ambient state. Silently accepting a missing pop or named frame was rejected in favor of a transactional `CTOOL_C_PP_DIAG_PRAGMA_PACK` error. An unmatched final push remains accepted: GCC 15 accepts it while Clang 22 diagnoses it only through a warning category, and the broader behavior avoids making valid host source artificially inexpressible. Unknown pragmas remain explicit unsupported directives. The active-source contract currently classifies exactly one `once` and two balanced `pack(push, 1)`/`pack(pop)` pairs at depth one; both FAT16 headers use those pairs for disk-wire layouts.

An active source-written `#exe` or `%:exe` in Cupid mode requires `{` as its first raw token on the same phase-two logical directive line. A macro cannot manufacture either the directive name or its opener. The output is one `CTOOL_C_PP_TOKEN_CUPID_EXE` token with spelling `exe`, the introducing `#`/`%:` location, and the current pack cap, followed by the ordinary expanded `{` and suffix tokens. Expansion may continue across physical lines until the next directive, so a function-macro call beginning on the directive tail can close on the following line. Later preprocessing directives retain their normal meaning rather than being captured as text. The preprocessor deliberately does not search for a matching brace: closure, nesting, placement, block ASTs, execution order, and JIT/AOT behavior belong to parser/backend policy. Active C11 use is an explicit unsupported-language error, malformed active Cupid use is an input error, inactive use has no effect, and every failure retains the operation's transactional result contract.

The checked active-source contract fails closed on conditional or non-block forms and currently records exactly one ordinary, unconditional block at `bin/feature6_exe.cc:7`. Focused contracts also prove digraph recognition, phase-two splicing, cross-line macro expansion, forced/include/once ordering, precise path/line/column/cap ownership, no hidden-function lowering, parser ownership of an unclosed block, and 256 markers without importing the private runtime's 128-block limit.

Source inspection of the private kernel frontend found why its current lowering is not the shared contract. JIT preprocessing rewrites blocks into generated `__cc_exe_N` functions and later discovers them by symbol-prefix scan; AOT skips them with a warning. Its capture limit is 128, brace counting does not model block comments, EOF capture is not diagnosed, capture can cross an include boundary, directives inside a captured block are suppressed, and a single-line AOT block can leave skip state active. The shared tape intentionally corrects the directive suppression and removes hidden names, prefix discovery, capture state, and execution limits from preprocessing. Parser/backend migration must use an explicit ordered representation instead of recreating those leaks. Whether AOT executes or only validates blocks, whether skipped bodies are type-checked, whether blocks are top-level-only or nestable, and whether header-origin markers are language-valid remain later language-policy decisions; preprocessing preserves header markers rather than deciding prematurely.

The build graph now produces one checked X-macro manifest of preprocessing jobs rather than hand-copying source paths into the C contract. Compile transforms must contain exactly one C root and one recognized recipe profile; Cupid delivery wrappers classify top-level programs, browser include-only fragments, and non-root delivered headers. New recipe variables, literal per-rule preprocessing flags, unknown Make functions, alternate include forms, unexpected source-consuming transforms, or cohort changes fail closed. Include-root order, forced inputs, explicit `-D` values, `-m32`, `-ffreestanding`, `-nostdinc`, and SSE selection come from GNU Make's evaluated values through dry-run database sentinels. Conditional and `:=` assignment semantics remain intact, and the Make graph stays authoritative instead of being reimplemented in Python.

Five source-driven request profiles cover the tracked OS-delivery corpus. Strict kernel, Doom compatibility, Doom-tree, and user C run in C11 mode with GNU extensions because their actual recipes use the GNU dialect; Cupid runtime mode does not opt into GNU preprocessing without source evidence. The i386 profiles receive explicit four-byte pointer and little-endian facts plus a stable `__GNUC__=1` compatibility marker; this marker means definedness and does not pretend to match Clang's or GCC's version. Kernel and Doom receive SSE2, only strict kernel receives `DEBUG`, and only Doom-tree receives its forced compatibility header and two configured port macros. Unused host/compiler aliases such as `__i386__` are deliberately absent from this minimal target contract until active source requires them.

Every tracked case opens a fresh job with a 256 MiB arena ceiling, loads one canonical root, runs the unchanged preprocessing operation, validates the public counted-tape invariants, and closes the job. The ceiling is generous and lazy, not a new library default: the dual-location hosted token is 80 bytes, so the 185,994-token generated ksyms tape needs 14,879,520 bytes for its final public token array alone, already beyond the default 8 MiB before raw caches and work nodes. The default gate covers 359 tracked roots: 152 kernel, six Doom compatibility, 80 Doom-tree, three user, 105 Cupid, twelve hosted Toolchain, and one hosted kernel bridge. A separate non-skipping target materializes and checks four generated kernel roots through the first-pass link. Twenty-two browser fragments are owned through `bin/browser.cc`. Two delivered headers retain an explicit missing standalone runtime context, while the remaining 19 hosted units are deferred only because they consume external system headers or runtime services.

Hosted preprocessing uses two exact profiles rather than one widened include search. `HOSTED_TOOLCHAIN_64` has only `/toolchain`; `HOSTED_KERNEL_BRIDGE_64` also has `/kernel/lang`, matching the one `as_elf` recipe that carries that path. Both select C11 without GNU extensions, set `hosted_environment`, and provide `__SIZEOF_POINTER__=8` as an explicit bootstrap-host fact. Omitting the pointer action was rejected because `ctool.c` would select its 32-bit address type on the current Windows AMD64 and Linux x86_64 hosts; deriving it from whichever machine runs the audit was rejected because the checked manifest must be deterministic. A future supported 32-bit bootstrap host therefore requires its own explicit profile and evidence rather than silently changing this one. The contract fails clearly when a `_64` profile runs in a process whose pointers are not eight bytes. Profile selection is also fail-closed at the compiler-argument seam: named profile metadata selects disjoint strict-hosted versus i386 modeled flags; every compile recipe has one direct compiler command with exact marker multiplicity and an ordered, unquoted preprocessor argument profile; the bridge include is pinned to its exact source recipes; and an external-runtime deferral remains valid only as a present tracked Toolchain host-object transform. Count-only profile totals, host-name inference, and path-only deferral were rejected because each could preserve the headline inventory while changing preprocessing semantics.

ADR 0013 supplies the downstream immutable indexed type graph and fixed i386 layout operation. Its member contract consumes the pack cap stamped on the first declaration-specifier token and retains both presumed and immutable physical locations in the graph; layout diagnostics use the presumed location as their primary source position. An aligned wrapper records the exact effective typedef/type-attribute alignment and may lower or raise natural alignment, while explicit record alignment only raises the computed record result. A separate layout-preserving `QUALIFIED` wrapper represents a qualified use of a nominal type without cloning its record/member slice and keeps `const T *` distinct from `T * const`. Atomic identity propagates through both wrappers, but alignment follows source order: a wrapper that introduces `_Atomic` applies the target minimum, a later exact `ALIGNED` wrapper may lower it, and a later non-atomic `QUALIFIED` wrapper preserves that effective alignment. Aggregate atomic qualification remains unsupported. Per-type `_Bool`, active-atomic, and atomic-minimum facts are cached during graph layout so member processing remains constant time and the complete operation remains `O(types + members)`. Layout enforces a flexible array's final structure position.

ADR 0014 now supplies the intervening declaration frontend. It consumes this tape without source rereads, applies declaration-time namespace, qualifier, completeness, flexible-array, storage, declarator, and target-integer rules, constructs the ADR 0013 graph, and invokes layout before publishing an immutable translation unit. Its unchanged `/kernel/fs/fat16.h` closure reproduces every manual FAT layout oracle; mutating only one already-expanded array-bound token changes the resulting layouts and proves the tape is authoritative. A legacy adapter around the private kernel frontend remains rejected because text re-lexing would discard counted spellings, columns, dual locations, pack state, and type identity while preserving direct parser-to-code generation.

Preprocessing profiles for the external-runtime hosted drivers and contracts, function bodies and initializers, typed AST/IR lowering, and kernel integration remain required before this tape replaces the private kernel preprocessor. The private kernel frontend still recognizes once and `#exe` independently and cannot preserve the typed tape's presumed/physical path, column, or pack metadata. The next gate is deepening the shared frontend against further unchanged active declarations and lowering them through the ADR 0003 path, not changing source to fit the old frontend.

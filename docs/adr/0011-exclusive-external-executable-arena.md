# Lease one permanently reserved arena for fixed external executables

Ordinary hosted-C/CupidLD executables use the identity-mapped interval `[0x00D00000, 0x00F00000)`. The PMM reserves the complete two-MiB interval permanently, while the process subsystem grants one generation-tagged exclusive lease for its contents. The loader accepts an external image only when every `PT_LOAD` byte fits inside this arena and its entry lies in file-backed bytes of an executable `PT_LOAD`. Successful process creation transfers the lease metadata into the PCB before publishing its PID; failure, exit, kill, reaping, or stack-canary cleanup cancels or releases exactly that lease and never releases the permanent pages.

The address is a compatibility bridge, not a general allocator. `0x00600000` was rejected after the measured kernel image showed that it overlaps initialized kernel state. The selected arena begins immediately after the fixed kernel stack and ends before CupidC's existing fixed region. `link.ld` separately asserts that the kernel memory image remains below the stack. The loader continues to accept CupidC `[0x01000000, 0x01900000)` and CupidASM `[0x01A00000, 0x01C00000)` AOT images as permanent legacy regions; they do not acquire the external lease and must never be returned to the PMM.

Lease release requires scheduler quiescence, not merely a terminated state. The outgoing CPU publishes `on_cpu = 0xFF` while holding the BKL, switches to the target ESP, restores the target FPU state, and only then releases the scheduler's sole BKL acquisition through the assembly/C handoff. The target's saved EFLAGS controls when interrupts resume. A switch attempted inside a nested BKL critical section becomes a CPU-local pending request and runs at the outer unlock instead of transferring lock ownership to another process. Remote kill publishes the target CPU's pending bit before sending the reschedule IPI; the real handler consumes it at the interrupt suspension point, or the target's outer BKL release consumes it if that CPU was already in a critical section. Reaping may free a stack and release its generation only after this detachment protocol completes.

The per-CPU selector is part of that execution context. Common exception and IRQ entry saves `%gs` but deliberately leaves the incoming per-CPU selector active while calling C; only `%ds`, `%es`, and `%fs` are changed to the flat kernel-data selector. Replacing `%gs` with the flat selector makes `this_cpu()` read linear address zero, so an interrupt-side BKL operation cannot identify its owner and may fault recursively before the original exception can be reported.

Reserving and releasing only each loaded page range was rejected because all external binaries share the same fixed addresses: PMM ownership alone cannot prevent two processes from overwriting one another, and releasing boot-reserved pages corrupts the allocator's ownership model. Arbitrary load addresses were rejected because they can overlap the growing kernel, stack, or fixed Cupid runtime regions. Position-independent or per-process-relocated executables are the eventual concurrency direction, but require corresponding compiler, linker, loader, and runtime semantics; they are not simulated by weakening validation in this transitional fixed-address path.

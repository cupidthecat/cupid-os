# ISO9660 Support in CupidOS

CupidOS mounts `.iso` files (ECMA-119 + Rock Ridge) as read-only
filesystems at arbitrary mount points.

## Quick start

1. Build a test ISO on the host with `make sync-iso`. The target uses
   `mkisofs -R` (or `genisoimage` / `xorrisofs`) to build
   `test_iso/hello.iso` from `test_iso/fixtures/`, then writes it directly to
   the FAT16 disk image as `/hello.iso`. Cupid OS exposes that file as
   `/disk/hello.iso`.
2. Boot CupidOS (`make run`).
3. In shell: `mount /disk/hello.iso /iso`
4. `ls /iso` lists `readme.txt`, `long_named_file.txt`, `sub/`, and `big.bin`.
5. `cat /iso/readme.txt` prints the content.
6. `umount /iso` when done.

## Features

- ECMA-119 base format with 8.3 uppercase filenames and `;1` versions.
- Rock Ridge (SUSP/RRIP) NM records provide long filenames. PX and TF are
  preserved where possible. The reader follows SUSP CE continuation chains up
  to eight levels.
- Path lookup is case-insensitive, so `/iso/README.TXT` matches
  `readme.txt`. Display names are preserved case.
- Up to four ISOs may be mounted at once.
- File operations include read, stat, and seek with `SEEK_SET`, `SEEK_CUR`, or `SEEK_END`.
- Directory enumeration uses the standard VFS `readdir` interface.

## Limitations

- The filesystem is read-only. Write, create, and unlink return `-VFS_EACCES`.
- There is no ATAPI driver. The source must be a `.iso` file accessible via VFS
  (e.g. on FAT16 at `/disk/...` or in homefs at `/home/...`).
- Joliet is not supported. Unicode-only ISOs made on Windows use base names in 8.3 format.
- Symlink traversal is not supported. Rock Ridge `SL` records are ignored.
- Deep-tree relocation with CL, PL, or RE records is not supported. Use `mkisofs -D 1`
  (the default) to produce ISOs that don't need them.
- Files are limited to 4 GB by the ECMA-119 32-bit LBA format.

## Architecture

```
shell `mount` ──► vfs_mount ──► iso9660_vfs (adapter, 4-slot pool)
                                     │
                                     ▼
                                iso9660.c  (PVD / dir records / SUSP)
                                     │
                                     ▼
                                loopdev.c  (file-backed block_device_t)
                                     │
                                     ▼
                                blockcache (existing LRU sector cache)
                                     │
                                     ▼
                                vfs_read on the .iso file
```

## Testing

- `test_iso/fixtures/` - checked-in source tree.
- `make sync-iso` - rebuilds hello.iso and syncs into the FAT partition.
- `bin/feature17_iso.cc` - end-to-end smoke test.
  Run `feature17_iso` from the shell; expect `PASS feature17_iso`.

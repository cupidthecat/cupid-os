# ISO9660 Support in CupidOS

CupidOS mounts `.iso` files (ECMA-119 + Rock Ridge) as read-only
filesystems at arbitrary mount points.

## Quick start

1. Host-side: build a test ISO:
   `make sync-iso` — generates `test_iso/hello.iso` via `mkisofs -R`
   (or `genisoimage` / `xorrisofs`) from `test_iso/fixtures/` and
   mcopies it to `/disk/hello.iso`.
2. Boot CupidOS (`make run`).
3. In shell: `mount /disk/hello.iso /iso`
4. `ls /iso` → shows `readme.txt`, `long_named_file.txt`, `sub/`, `big.bin`.
5. `cat /iso/readme.txt` → prints the content.
6. `umount /iso` when done.

## Features

- **ECMA-119 base format** (8.3 uppercase filenames, `;1` versions).
- **Rock Ridge (SUSP/RRIP)**: NM records for long filenames, PX/TF
  preserved where possible. SUSP CE continuation chains followed up
  to 8 levels.
- **Case-insensitive path lookups**: `/iso/README.TXT` matches
  `readme.txt`. Display names are preserved case.
- **Multi-mount**: up to 4 ISOs mounted concurrently.
- **File I/O**: read, seek (SEEK_SET/CUR/END), stat.
- **readdir** via the standard VFS interface.

## Limitations

- **Read-only**. Write/create/unlink return `-VFS_EACCES`.
- **No ATAPI driver**. Source must be a `.iso` file accessible via VFS
  (e.g. on FAT16 at `/disk/...` or in homefs at `/home/...`).
- **No Joliet**. Windows-made Unicode-only ISOs use base names (8.3).
- **No symlink traversal**. Rock Ridge `SL` records are ignored.
- **No deep-tree relocation** (CL/PL/RE records). Use `mkisofs -D 1`
  (the default) to produce ISOs that don't need them.
- **4 GB filesize limit** (ECMA-119 32-bit LBA).

## Architecture

```
shell `mount` ──► vfs_mount ──► iso9660_vfs (adapter, 4-slot pool)
                                     │
                                     ▼
                                iso9660.c  (PVD / dir records / SUSP)
                                     │
                                     ▼
                                loopdev.c  (file-backed block_device_t)
                                     │
                                     ▼
                                blockcache (existing LRU sector cache)
                                     │
                                     ▼
                                vfs_read on the .iso file
```

  Run `feature17_iso` from the shell; expect `PASS feature17_iso`.
